# FlareVM: Arsenal of Tools

## What FlareVM Is

- A Windows VM toolkit curated by FireEye’s FLARE team for reverse engineering, malware analysis, DFIR, and pentesting.
- Provides a one-stop “toolbox” for static/dynamic analysis, debugging, forensics, and networking.

## Tool Categories & Standouts

### **Reverse Engineering & Debugging**

- Ghidra, x64dbg, OllyDbg, Radare2, Binary Ninja, PEiD — disassembly/decompilation, interactive debugging, packer/compiler detection.

### **Disassemblers & Decompilers**

- CFF Explorer (PE editor), Hopper, RetDec — structure inspection and code recovery.

### **Static & Dynamic Analysis**

- Process Hacker, PEview, Dependency Walker, Detect It Easy (DIE) — metadata, imports, packer hints, dependency graphs.

### **Forensics & IR**

- Volatility, Rekall, FTK Imager — memory and disk evidence acquisition/analysis.

### **Network Analysis**

- Wireshark, Nmap, Netcat — traffic capture, host discovery, on-the-wire triage.

### **File Analysis**

- FileInsight, Hex Fiend, HxD — byte-level inspection and edits.

### **Scripting & Automation**

- Python, PowerShell Empire — automation and scripted post-exploitation.

### **Sysinternals (core triage)**

- Autoruns, Process Explorer, Process Monitor — autostarts, process internals, real-time system activity.

## “Commonly Used” Investigation Baseline

- **Procmon**: file/registry/process activity timeline for troubleshooting & malware behavior.
- **Process Explorer**: parent/child tree, handles, DLLs, path, network (TCP/IP tab).
- **HxD**: hex/ASCII view to spot magic bytes, strings, and modify/test safely.
- **Wireshark**: protocol-level view; detect suspicious flows/hosts.
- **CFF Explorer**: PE headers, imports, hash calculation.
- **PEStudio**: static triage (entropy, imports, indicators) without execution.
- **FLOSS**: extract/deobfuscate strings (static, stack, tight); quick IOC discovery.

## How to Use the Core Tools (Quick Patterns)

- **Procmon**
    - Start capture → set filters (Process Name → contains → *target*) → review File/Reg/Process operations.
    - Look for abnormal file writes, registry autoruns, access to `lsass.exe`/sensitive paths.
- **Process Explorer**
    - Find process → verify **Parent** (e.g., `Explorer.exe` spawning doc/LNK/ISO payloads).
    - Check **Properties → TCP/IP** for live destinations/ports and **Strings/Handles/DLLs** for hints.
- **HxD**
    - Open sample → confirm file type via magic (`4D 5A` = PE) → use **Data Inspector** for typed views.
- **CFF Explorer**
    - Review headers, sections, imports; compute **MD5/SHA-1**; compare against VT/known-good baselines.
- **Wireshark**
    - Capture/inspect → filter by process/port/IP; note TLS usage that may hide C2.
- **PEStudio**
    - Check **entropy**, **imports** (blacklist view), **version info**, **rich header**, **path anomalies**.
- **FLOSS**
    - Run against binary → harvest static/decoded strings → pivot to URLs, domains, API usage.

## Case Study 1: `windows.exe` (Static Triage)

- **Hashes**: MD5 `9FDD4767DE5AEC8E577C1916ECC3E1D6`; SHA-1 `A1BC55A7931BFCD24651357829C460FD3DC4828F` (submit to VT for reputation).
- **Impersonation**: Claims to be *Windows Registry Editor*; stored in user **Downloads/Sample** (not `C:\Windows\System32` → suspect).
- **Locale Mismatch**: Russian metadata (“Редактор реестра”, “Microsoft® Windows®”) without org context → red flag.
- **Pack/Obfuscate Hint**: Missing **rich header**.
- **Imports/Behavioral Clues**:
    - `set_UseShellExecute` → shell-spawned child processes (process injection/launch).
    - `CryptoStream`, `RijndaelManaged`, `CipherMode`, `CreateDecryptor` → AES use (config/C2/file crypto; possible ransomware/secure comms).
- **FLOSS Result**: Static strings align with PEStudio’s imports; no decoded strings (likely .NET/obfuscation—FLOSS warns .NET deobfuscation not supported).

### **Takeaways**

- Treat as likely packed/obfuscated .NET sample impersonating `regedit`; expect process-spawning and encrypted payload/traffic. Proceed to sandboxed dynamic analysis.

## Case Study 2: `cobaltstrike.exe` (Dynamic Net Triage)

- **Process Explorer**
    - Parent/child confirms `Explorer.exe` → `cobaltstrike.exe`; PID example: **4756**.
    - **TCP/IP tab** shows live destination(s) and state.
- **Procmon**
    - Apply filter: *Process Name contains cobalt*; correlate socket operations and outbound connections.
- **Outcome**
    - Confirmed outbound connectivity to **47.120.46.210** (unknown host) consistent with potential C2 behavior.

## Practical Workflow (Start-to-Finish)

1. **Hash & Signatures**: Compute MD5/SHA-1; check reputation (VT). Note signer/metadata vs. expected path.
2. **PE Triage**: PEStudio/CFF Explorer for entropy, sections, imports, rich header, blacklisted APIs.
3. **Strings/Config**: FLOSS for quick IOCs (URLs, keys, mutexes). If .NET, consider dnSpy/ILSpy (noted limitation of FLOSS).
4. **Behavior**: Execute in isolated VM → Process Explorer (parent/child, TCP/IP), Procmon (file/reg/process).
5. **Network**: Wireshark to validate destinations/protocols; pivot on IPs/domains.
6. **Decide**: If impersonation, crypto use, and C2 observed → treat as malicious; document IOCs and containment steps.

## Key IOCs & Clues from Examples

- **Impersonation & Path**: “Registry Editor” binary outside `System32`.
- **Metadata Mismatch**: Unexpected Russian locale.
- **Packing**: Missing rich header; medium-high entropy (PEStudio).
- **Crypto Imports**: AES (Rijndael) usage in .NET.
- **Network**: Outbound to **47.120.46.210** from `cobaltstrike.exe`.

## Quick Reference (Commands/Clicks)

- **Procmon**: `Ctrl+L` → *Process Name contains* → include → Apply.
- **Process Explorer**: Right-click process → **Properties → TCP/IP**.
- **FLOSS**: `FLOSS.exe .\sample.exe > out.txt` (note: limited for .NET).
- **Wireshark**: Filter examples — `tcp`, `tls`, `ip.addr == <suspect IP>`.

## **Bottom line**

FlareVM centralizes the essentials for triage: verify identity (hashes/metadata/path), inspect PE structure/imports, extract strings for IOCs, and corroborate behavior with Procmon/Process Explorer/Wireshark. In the examples, indicators (impersonation, AES usage, missing rich header) and confirmed outbound traffic point toward malicious activity and likely C2 communication.
