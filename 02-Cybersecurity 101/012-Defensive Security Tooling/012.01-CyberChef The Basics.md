# CyberChef: The Basics

## 🧰 Overview

**CyberChef** is a **web-based data analysis and transformation tool** — often called the *“Swiss Army knife for data”*.

It performs various **encoding, decoding, encryption, and data manipulation** tasks using a sequence of actions called **recipes**.

## 🔗 Access Methods

- **Online:**
    
    Access directly via browser: [https://gchq.github.io/CyberChef](https://gchq.github.io/CyberChef)
    
- **Offline:**
    
    Download from [GitHub Releases](https://github.com/gchq/CyberChef/releases) and run locally (Windows/Linux).
    

## 🧭 Interface Structure

CyberChef consists of **four main areas**:

1. **Operations Area**
    - Repository of all available functions, searchable and categorized.
    - Example operations:
        - `From Morse Code` → translates Morse to text
        - `URL Encode` / `Decode` → handles percent-encoded URLs
        - `To Base64`, `To Hex`, `To Decimal`, `ROT13`, etc.
2. **Recipe Area**
    - Core workspace to **assemble and configure operations**.
    - Functions: *Save*, *Load*, *Clear* recipes, and process via **“BAKE!”** or **Auto Bake**.
3. **Input Area**
    - For text/file/folder input.
    - Features: *Add tab*, *Open file/folder*, *Clear input/output*, *Reset layout*.
4. **Output Area**
    - Displays processed data.
    - Features: *Save output*, *Copy to clipboard*, *Replace input with output*, *Maximize view*.

## 🧩 Workflow: “CyberChef Thought Process”

1. **Define Objective** – What do you want to accomplish? (e.g., decode a suspicious string)
2. **Input Data** – Paste or upload content into the input area.
3. **Select Operations** – Choose appropriate transformations (e.g., `ROT13`, `Base64`, `Base85`).
4. **Check Output** – Validate if the result meets the objective; adjust if not.

## ⚙️ Common Operation Categories

### **1. Extractors**

- `Extract IP addresses` – Finds all IPv4/IPv6 patterns.
- `Extract URLs` – Pulls URLs (requires valid protocol).
- `Extract email addresses` – Detects patterns like `name@domain.com`.

### **2. Date / Time**

- `From UNIX Timestamp` – Converts UNIX time to readable date.
- `To UNIX Timestamp` – Converts date to UNIX epoch (seconds since Jan 1, 1970 UTC).

### **3. Data Format**

- **Base Encodings:**
    - `From Base64` – Decodes Base64 → raw text.
    - `From Base85`, `From Base58`, `To Base62` – Similar encoding variations for efficient text representation.
- **URL Decode** – Converts `%`encoded values (UTF-8) back to readable characters (e.g., `%3A` → `:`).

## 🧮 Manual Base64 Encoding Example

Encoding “**THM**” manually:

1. **Convert to Binary (ASCII):**
    
    `T=01010100`, `H=01001000`, `M=01001101` → Combined: `010101000100100001001101`
    
2. **Split into 6-bit chunks:**
    
    `010101` `000100` `100001` `001101` → Decimal: 21, 4, 33, 13
    
3. **Map via Base64 Index Table:**
    
    21 → V, 4 → E, 33 → h, 13 → N
    
    **Result:** `VEhN`
    

Thus, `"THM"` in Base64 = **`VEhN`**.

## 🔑 Key Takeaways

- CyberChef simplifies **data transformation, encoding, decoding, and extraction** without coding.
- It operates through **recipes**—chained operations applied in sequence.
- Understanding **common data formats (Base64, URL encoding, timestamps, etc.)** is essential for effective use.
- The tool supports both **online and offline** use, making it highly accessible for cybersecurity analysis and forensics.
