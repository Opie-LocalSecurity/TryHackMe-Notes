# Gobuster: The Basics

## **Purpose & place in workflow**

- Gobuster is a fast, Go-based brute-forcing tool for enumerating **web directories/files**, **DNS subdomains**, **VHOSTs**, **AWS S3**, **GCS**, and a few other targets.
- Typically used between **reconnaissance** and **scanning** in a pentest/bug-bounty workflow.

## **Core concepts**

- **Enumeration**: list available resources (even if inaccessible).
- **Brute force**: try entries from a wordlist (each entry becomes an attempted path, subdomain, or host header value).
- Gobuster operates by constructing requests from wordlist entries and reporting responses (status codes, sizes, IPs, etc.).

## **Main modes (commands)**

- `dir` ‚Äî directory/file enumeration (HTTP GET requests to base URL + each wordlist entry).
- `dns` ‚Äî DNS subdomain brute force (construct FQDNs and perform DNS lookups).
- `vhost` ‚Äî virtual host brute force (change `Host:` header to test name on an IP-hosted webserver).
- (also `fuzz`, `s3`, `gcs`, `tftp`, etc.)

## **Important global flags**

- `w, --wordlist` ‚Äî path to wordlist (required).
- `t, --threads` ‚Äî concurrency (default 10).
- `-delay` ‚Äî pause between requests (throttle to avoid detection/rate-limit).
- `o, --output` ‚Äî write results to file.
- `-wordlist-offset` ‚Äî resume from a wordlist position.
- `-debug / -v / -q` ‚Äî debug/verbose/quiet modes.
- `-no-color`, `-no-progress`, `-no-error` ‚Äî output control.

## **Key `dir` flags & behaviour**

- `u` (URL) and `w` required.
- `x, --extensions` ‚Äî append file extensions (e.g. `.php,.js`) to each word.
- `s, --status-codes` and `b, --status-codes-blacklist` ‚Äî control which HTTP statuses to show.
- `H, --headers` and `c, --cookies` ‚Äî send custom headers/cookies (useful for authenticated or custom requests).
- `k, --no-tls-validation` ‚Äî ignore TLS cert errors.
- `r, --follow-redirect` ‚Äî follow HTTP redirects.
- **Note:** Gobuster `dir` is **not recursive** ‚Äî if you find `/admin/` you must run another scan targeting `/admin/`.

## **Key `dns` flags & behaviour**

- `d, --domain` and `w` required.
- `i, --show-ips` ‚Äî display resolved IPs.
- `c, --show-cname` ‚Äî show CNAMEs.
- `r, --resolver` ‚Äî use a custom DNS resolver.

## **Key `vhost` flags & behaviour**

- `u` (URL/IP) and `w` required. `-domain` and `-append-domain` frequently needed when DNS isn‚Äôt configured.
- Changes the **Host** header for each wordlist entry ‚Äî useful for sites hosting many vhosts on one IP.
- `-exclude-length` ‚Äî filter results by response body length to reduce false positives (common technique).
- `m, --method` ‚Äî set HTTP method (GET/POST etc).

## **Practical tips & pitfalls**

- Prefer using **hostname** rather than raw IP when targeting a specific site; using IP may target a different vhost. For `vhost` scans you often use IP + `-domain`/`-append-domain`.
- Use `-delay` and lower thread counts against production targets to avoid lockouts and detection.
- Use `-exclude-length` / status code filters to cut down false positives (many 404 pages have identical sizes).
- Resume scans with `-wordlist-offset` to continue long jobs.
- Save results with `o` for later parsing/triage.
- Use authenticated headers/cookies when you have credentials to enumerate restricted areas.

## **Representative commands**

- Directory scan (basic): `gobuster dir -u "http://example.thm/" -w /path/to/wordlist -t 64`
- Directory scan with extensions and redirects: `gobuster dir -u "http://example.thm/" -w /usr/share/wordlists/directory-list.txt -x .php,.js -r`
- DNS subdomain enumeration: `gobuster dns -d example.thm -w /usr/share/wordlists/subdomains.txt`
- VHost enumeration (IP target + append domain + filter sizes): `gobuster vhost -u "http://10.10.10.10" --domain example.thm --append-domain -w subdomains.txt --exclude-length 250-320`

## **Outcome / value**

- Quickly discovers hidden paths, files, subdomains, and virtual hosts that can reveal attack surface and vulnerable endpoints ‚Äî a high-value early step in offensive assessments.
