# Hydra

## **What it is**

- Hydra is a fast, online brute-force authentication tool for testing/login cracking across many protocols.
- Intended for security testing (red-team / pentest) — use **only** on systems you own or are authorized to test.

## **Key points / outcomes**

- Supports a very large list of services (SSH, FTP, many HTTP variants, SMB, RDP, VNC, databases, SNMP, etc.).
- Demonstrates why weak/default passwords (e.g. `admin:password`) are dangerous — use long, unique, complex passwords and change defaults.

## **Common usage patterns**

- **SSH brute force (single username + password list):**
    - `hydra -l <username> -P <wordlist> <target_ip> -t <threads> ssh`
    - Important flags: `l` (single user), `P` (password list), `t` (threads).
    - Example: `hydra -l root -P passwords.txt 10.0.0.5 -t 4 ssh`
- **HTTP form (POST) brute force:**
    - `hydra -l <user> -P <wordlist> <target> http-post-form "<path>:<post-data>:<failure-string>" -V`
    - `^USER^` and `^PASS^` are replaced by Hydra with the username/password being tried.
    - You must supply the **exact** form path, parameter names and a string that appears on failed login responses (used to detect failure vs success).
    - Example: `hydra -l admin -P passlist.txt 10.0.0.5 http-post-form "/login.php:username=^USER^&password=^PASS^:F=incorrect" -V`

## **Practical tips**

- Use the browser dev tools (Network tab) or view page source to determine:
    - whether the form uses GET or POST,
    - the correct form action path,
    - parameter names for username/password,
    - failure message or status returned on bad login.
- Tune `t` (threads) to balance speed vs target/server stability and detection risk.
- Use verbosity `V` for per-attempt output, or quieter modes for large runs.
- Large password lists (millions) increase success chance for weak creds but also increase noise and detection risk.

## **Important flags (summary)**

- `l <user>` — single username
- `L <file>` — file of usernames
- `P <file>` — file of passwords
- `t <n>` — number of parallel threads
- `V` — verbose (show each attempt)
- `http-post-form` / `http-get` / protocol name — specify service type

## **Ethics & safety**

- Using Hydra against systems you don’t own or have explicit permission to test is illegal and unethical.
- Use it only for authorized penetration tests, lab exercises, or defensive assessments. Log and scope tests, and avoid causing denial-of-service on targets.
