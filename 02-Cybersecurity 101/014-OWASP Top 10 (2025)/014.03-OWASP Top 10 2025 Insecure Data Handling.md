# OWASP Top 10 2025: Insecure Data Handling

## **A04: Cryptographic Failures**

### **What It Is**

- Sensitive data is exposed due to weak, missing, or incorrectly implemented cryptography.
- Common causes:
    - Storing passwords without hashing
    - Using outdated/weak algorithms (MD5, SHA-1, DES)
    - Exposing encryption keys
    - Not encrypting data in transit
    - “Rolling your own crypto” instead of using proven libraries

### **How to Prevent It**

- Use strong, modern, vetted algorithms and libraries.
- Hash passwords with slow, adaptive functions: **bcrypt, scrypt, Argon2**.
- Do *not* write custom cryptographic algorithms.
- Protect secrets: avoid embedding keys in code or configs; use secure key-management systems.

## **A05: Injection**

### **What It Is**

- Occurs when untrusted input is fed directly into interpreters (SQL engine, shell, template engine, API, etc.).
- Classic forms:
    - **SQL Injection**
    - **Command Injection**
    - **Server-Side Template Injection (SSTI)**
    - **Prompt injection (AI)**
- Still highly prevalent; appears multiple times across OWASP Top 10 versions.

### **How to Prevent It**

- Always treat user input as untrusted.
- Use **prepared statements / parameterized queries** for SQL.
- Avoid shell-invoking functions; use safe APIs instead.
- Apply robust input validation:
    - Enforce strict data types
    - Escape dangerous characters
    - Validate before processing

## **A08: Software or Data Integrity Failures**

### **What It Is**

- Happens when an application trusts code, updates, or data without verifying authenticity or integrity.
- Examples:
    - Unverified software updates
    - Loading scripts/configs from untrusted sources
    - Accepting altered binaries, templates, or JSON files
    - Not validating critical data that influences logic

### **How to Prevent It**

- Establish and enforce **trust boundaries**.
- Verify integrity before use: checksums, digital signatures, cryptographic validation.
- Ensure only trusted sources can modify critical artifacts.
- Integrate integrity checks into development and **CI/CD pipelines**.
