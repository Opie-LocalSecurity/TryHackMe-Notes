# NMAP: The Basics

## Target Specification

- Ranges: `192.168.0.1-10`
- CIDR: `192.168.0.1/24` (≅ `192.168.0.0-255`)
- Hostnames: `example.thm`

## Host Discovery (Who is online?)

- Basic “ping sweep”: `sn` (find live hosts, don’t scan ports)
- Local/L2 network (same Wi-Fi/Ethernet):
    - Uses ARP requests; returns MAC + vendor (handy to infer device types).
- Remote network (across one+ routers):
    - Uses ICMP echo/timestamp + TCP SYN/ACK and UDP probes to infer liveness.
- Control probe types (advanced): `PS[ports]` (TCP SYN), `PA[ports]` (TCP ACK), `PU[ports]` (UDP).
- List targets only (no packets sent to them): `sL` (sanity-check scope before scanning).

## Port Scanning (Who is listening?)

- TCP connect scan `sT`: completes full 3-way handshake for each port; works unprivileged.
- TCP SYN (stealth) `sS`: sends SYN only; no full connection; fewer logs; requires root.
- UDP scan `sU`: discovers UDP services (DNS/DHCP/NTP/SNMP/VoIP). Closed UDP often replies with ICMP “port unreachable”; silence ≠ open.
- Limit ports:
    - Fast 100 common ports: `F`
    - Specific range/list: `p10-1024`, `p80,443,8080`, all ports: `p-` (1–65535)
- Defaults: Nmap scans the 1,000 most common TCP ports unless you override.

## Service & OS Fingerprinting (Extract more info)

- OS guess: `O` (e.g., “Linux 4.x|5.x”)
- Service & version: `sV` (adds a VERSION column)
- “All the things”: `A` (enables `O`, `sV`, traceroute, and more)
- Force port scan even if host discovery failed (e.g., ICMP blocked): `Pn`

## Timing & Performance (How fast is fast?)

- Timing templates: `T0..-T5` → paranoid(0), sneaky(1), polite(2), normal(3), aggressive(4), insane(5)
    - Slower timings insert larger delays; good for evasion/fragile links.
- Parallelism: `-min-parallelism N`, `-max-parallelism N` (active probes per host group)
- Packet rate (global): `-min-rate N`, `-max-rate N` (packets/sec)
- Give up on slow hosts: `-host-timeout 30m` (example)

## Output & Logging (Controlling what you see)

- Verbosity: `v`, `vv`, `v4` (can press `v` during scan)
- Debug: `d`, …, `d9` (very chatty)
- Save reports:
    - Normal: `oN file`
    - XML: `oX file`
    - Grepable: `oG file`
    - All three: `oA basename` → `basename.nmap/.xml/.gnmap`

## Privileges Matter

- Run with `sudo` to unlock raw-packet scans (e.g., `sS`, ARP).
- Unprivileged runs fall back to `sT` and simpler host discovery.

## Handy One-Liners

- Ping sweep local /24: `sudo nmap -sn 192.168.66.0/24`
- Enumerate hosts then full TCP+UDP top ports:
    
    `sudo nmap -sn 10.10.0.0/16 -oG live.gnmap` → feed live IPs to next step
    
    `sudo nmap -sS -sU -F -sV -O -oA quick  -iL <(awk '/Up$/{print $2}' live.gnmap)`
    
- Full TCP scan (all ports) with versions: `sudo nmap -sS -p- -sV -oA fulltcp target`
- Stealthy, slow scan of top 100: `sudo nmap -sS -F -T1 target`
- Force-scan firewall-hidden hosts: `sudo nmap -Pn -sS -p- target`
- Save everything neatly: `... -oA projectname`

## At-a-Glance Option Map

- **Discovery**: `sn`, `sL`, `PS/PA/PU`
- **Ports**: `sT`, `sS`, `sU`, `F`, `p[range]`, `p-`
- **Fingerprinting**: `sV`, `O`, `A`, `Pn`
- **Speed**: `T0..5`, `-min/max-parallelism`, `-min/max-rate`, `-host-timeout`
- **Output**: `v/-vv/-d`, `oN/-oX/-oG/-oA`

### **Mental model**:

1. **Find hosts** (`sn` / ARP vs. ICMP/TCP/UDP).
2. **Find ports** (`sS`/`sT`/`sU`, choose scope with `p`).
3. **Fingerprint** (`sV`, `O`, or `A`).
4. **Tune** (timing/parallelism/rate).
5. **Report** (verbosity + `oA`).
