# Networking Secure Protocols

## **TLS (Transport Layer Security)**

- **Purpose:** Ensures **confidentiality and integrity** of data between client and server over insecure networks.
- **History:**
    - **SSL 2.0 (1995)** → developed by Netscape.
    - **TLS 1.0 (1999)** → IETF upgrade to SSL 3.0.
    - **TLS 1.3 (2018)** → major modernization, improved efficiency and security.
- **How it works:**
    - Operates at the **Transport Layer** (OSI model).
    - Uses **encryption, authentication, and integrity checks**.
    - Requires a **digital certificate** signed by a **Certificate Authority (CA)**.
    - **Let’s Encrypt** provides free CA-signed certificates.
    - **Self-signed certificates** offer encryption but not authenticity.
- **Adoption:** Added to many protocols → HTTP → **HTTPS**, DNS → **DoT**, MQTT → **MQTTS**, SIP → **SIPS**.

## **HTTPS (HTTP over TLS)**

- **HTTP Basics:**
    - Uses **TCP port 80**.
    - Data is **sent in cleartext**, easy to intercept.
    - Communication:
        1. TCP handshake
        2. HTTP requests (e.g., `GET / HTTP/1.1`)
- **HTTPS Process:**
    - Uses **TCP port 443**.
    - Communication steps:
        1. TCP handshake
        2. TLS handshake (session establishment)
        3. Encrypted HTTP exchange
- **Result:**
    - Traffic appears as “Application Data” in Wireshark; actual content is **encrypted**.
    - **Decryption** possible only with the session’s **private key**.
- **Key takeaway:** TLS adds security **without altering** HTTP, TCP, or IP protocols.

## **SMTPS, POP3S, and IMAPS**

- **Concept:** Same principle as HTTPS — adding TLS to secure existing email protocols.
- **Default Ports:**
    
    
    | Protocol | Insecure Port | Secure Port |
    | --- | --- | --- |
    | HTTP | 80 | 443 |
    | SMTP | 25 | 465 / 587 |
    | POP3 | 110 | 995 |
    | IMAP | 143 | 993 |
- **Outcome:** Encryption and integrity protection for email communication.

## **SSH (Secure Shell)**

- **Origin:** Created by **Tatu Ylönen** in 1995 as a secure alternative to Telnet (port 23).
- **Successors:**
    - **SSH-2 (1996)** – improved version.
    - **OpenSSH (1999)** – open-source implementation widely used today.
- **Key Features:**
    - **Authentication:** Password, public key, or multi-factor.
    - **Confidentiality & Integrity:** End-to-end encryption, protection against MITM.
    - **Tunneling:** Securely forwards other protocols (VPN-like).
    - **X11 Forwarding:** Allows remote GUI applications.
- **Usage:** `ssh username@hostname` (default port **22**).

## **SFTP and FTPS**

- **SFTP (SSH File Transfer Protocol):**
    - Uses **SSH** (port 22) for encrypted file transfers.
    - Commands: `get filename`, `put filename`.
    - Part of the **SSH suite**.
- **FTPS (FTP Secure):**
    - **FTP + TLS** (port 990).
    - Requires proper TLS certificate.
    - More complex with firewalls due to separate control/data channels.
- **Key Difference:**
    - **SFTP** → secured via SSH
    - **FTPS** → secured via TLS

## **VPN (Virtual Private Network)**

- **Purpose:** Create a **private, encrypted tunnel** over the public Internet for secure remote connectivity.
- **Use Cases:**
    - Connect multiple offices or remote users to a central network.
    - Provide access to internal resources securely.
- **Operation:**
    - **VPN client** ↔ **VPN server** → encrypts traffic via a **VPN tunnel**.
    - Once connected, **all traffic** can be routed through the tunnel.
- **Benefits:**
    - Hides user’s public IP (appears as VPN server’s).
    - Bypasses censorship or geo-restrictions.
    - Protects against ISP monitoring.
- **Cautions:**
    - Some VPNs **leak DNS/IP info** or only partially route traffic.
    - **Legal restrictions** exist in some countries.

## **Overall Summary**

- **TLS** revolutionized Internet security by providing **encryption and authentication** for numerous protocols.
- **HTTPS, SMTPS, IMAPS, POP3S, FTPS** — all use **TLS** for secure communication.
- **SSH/SFTP** — alternative secure channel using **SSH encryption**.
- **VPNs** — extend privacy and security across entire networks via encrypted tunnels.

**Core Principle:** Encryption and authentication are foundational to modern Internet security — TLS, SSH, and VPNs are the main pillars enabling safe digital communication.
