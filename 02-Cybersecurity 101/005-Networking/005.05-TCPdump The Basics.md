# TCPdump: The Basics

# Core workflow

- Pick interface: `i <iface>` (e.g., `eth0`, `wlo1`) or all: `i any`.
- Save capture for later: `w <file.pcap>` (no live printout when writing).
- Read a capture: `r <file.pcap>` (can add filters/formatting while reading).
- Limit volume: `c <COUNT>` to stop after N packets.
- Speed up/keep raw values: `n` (no DNS), `nn` (no DNS or service name).
- Verbosity: `v`, `vv`, `vvv` add header details.

**Examples**

- `tcpdump -i eth0 -c 50 -v` – 50 verbose packets on `eth0`.
- `tcpdump -i wlo1 -w data.pcap` – write to file until Ctrl-C.
- `tcpdump -i any -nn` – show all interfaces, no name/port resolution.

## Display controls (how packets are printed)

- `q` quick/brief lines.
- `e` include link-layer (MAC) headers.
- `A` payload as ASCII (good for plaintext).
- `xx` payload + headers in hex.
- `X` hex + ASCII together.

## Filtering (what packets to include)

Use expressions after options; combine with `and`, `or`, `not`.

### By host

- Any direction: `host <IP|NAME>`
- Source only: `src host <IP|NAME>`
- Destination only: `dst host <IP|NAME>`

### By port

- Any direction: `port <N>`
- Source: `src port <N>`
- Destination: `dst port <N>`

### By protocol

- `ip`, `ip6`, `tcp`, `udp`, `icmp`, etc.
    
    Example: `tcpdump -i any tcp port 22` (SSH),
    
    `tcpdump -i wlo1 udp port 123` (NTP).
    

### Logical operators

- `and` (both true), `or` (either true), `not` (negate).
    
    Example: `tcpdump -i eth0 host example.com and tcp port 443 -w https.pcap`.
    

## Advanced filtering (pcap-filter syntax)

Target specific header bytes with `proto[expr[:size]]`, where:

- `proto` = `arp|ether|ip|ip6|tcp|udp|icmp`, etc.
- `expr` = byte offset (0-based), `size` = 1/2/4 bytes (default 1).

### TCP flag filters (most practical)

Refer to flags via `tcp[tcpflags]` and test with bitwise ops:

- Only SYN set: `"tcp[tcpflags] == tcp-syn"`
- At least SYN set: `"tcp[tcpflags] & tcp-syn != 0"`
- SYN or ACK set: `"tcp[tcpflags] & (tcp-syn|tcp-ack) != 0"`

### Size filters

- `greater <LEN>` – length ≥ LEN
- `less <LEN>` – length ≤ LEN

## Typical review commands

- First 5 packets from file, no lookups: `tcpdump -r traffic.pcap -c 5 -n`
- Count matches (via shell): `tcpdump -r traffic.pcap src host 192.168.124.1 -n | wc -l`

## Key takeaways

- Always scope: interface + filters; otherwise output is noisy and slow.
- Write to `.pcap` for repeatable, tool-agnostic analysis (Wireshark, etc.).
- Use `nn` for speed/accuracy during triage; add `v` when you need header details.
- Combine filters logically to isolate exactly the traffic you care about (host/port/proto/flags/size).
- Choose display mode (`A`, `xx`, `X`, `e`, `q`) to suit the inspection task.
