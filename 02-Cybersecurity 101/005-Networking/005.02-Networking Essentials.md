# Networking Essentials

## DHCP — Auto-configuring hosts

- **Purpose:** Automatically assign IP settings (IP + mask, default gateway, DNS) to clients.
- **Transport:** **UDP** — server listens on **67**, client sends from **68**.
- **Workflow (DORA):**
    - **Discover:** client broadcasts from `0.0.0.0` → `255.255.255.255`.
    - **Offer:** server proposes an IP/lease + options.
    - **Request:** client asks to use the offered IP.
    - **Ack:** server confirms; lease becomes active.
- **Observations from pcap:**
    - Early client frames use **broadcast MAC `ff:ff:ff:ff:ff:ff`** and **IP `0.0.0.0`**.
    - Server unicasts/broadcasts the offer/ack including gateway & DNS options.
- **Outcome:** Client gets a **leased IP**, **gateway**, **DNS** → ready for LAN/Internet.

## ARP — L3→L2 address resolution

- **Problem:** To send an IP packet on Ethernet/Wi-Fi, you need the **destination MAC**.
- **Mechanism:** **ARP Request** (broadcast MAC) asks “Who has IP X?” → **ARP Reply** (unicast) “IP X is at MAC Y.”
- **Encapsulation:** **Directly in Ethernet** (not IP/UDP/TCP) — ARP is often called L2 (though it serves IP).
- **Result:** Sender caches IP↔MAC mapping; subsequent frames carry **Src/Dst MAC** with **Type=IPv4**.

## ICMP — Diagnostics & error reporting

- **Common tools:**
    - **`ping`** → ICMP **Echo Request (Type 8)** / **Echo Reply (Type 0)**; reports RTT & loss. Firewalls/host policy may drop ICMP.
    - **`traceroute` / `tracert`** → sends probes with increasing **TTL**; routers that drop at TTL=0 return **ICMP Time Exceeded (Type 11)**, revealing the path.
- **Notes:** Some hops don’t answer; paths can vary between runs.

## Routing — Moving packets across networks

- **Need:** Routers choose next hops so packets reach remote subnets/ASes.
- **You should recognize these protocols:**
    - **OSPF (link-state, interior):** Routers flood topology state; each computes shortest paths.
    - **EIGRP (Cisco, hybrid, interior):** Shares reachability + metrics (bandwidth/delay) to pick efficient paths.
    - **RIP (distance-vector, interior):** Chooses routes by **hop count** (simple, small networks).
    - **BGP (path-vector, exterior/Internet):** Exchanges routes between **autonomous systems** to steer inter-ISP traffic.

## NAT — Many private hosts, one public address

- **Motivation:** IPv4 scarcity; conserve public IPs.
- **How:** Router tracks flows in a **translation table** (internal `IP:port` ↔ external `IP:port`)—often called **PAT/NAPT**.
    - Example: `192.168.0.129:15401` ↔ `212.3.4.5:19273`.
- **Effect:** Multiple private hosts share a single public IP transparently for outbound connections.

## Practical commands to remember

- Capture/inspect DHCP/ARP: `tshark -r file.pcap -n`, `tcpdump -r file.pcap -n -v`
- Connectivity: `ping <ip-or-host> -c 4` (Linux/macOS)
- Path discovery: `traceroute <host>` (Linux/macOS) / `tracert <host>` (Windows)

## Key takeaways

- **DHCP** automates correct, conflict-free IP setup.
- **ARP** resolves **IP→MAC** so L3 packets can ride L2 frames.
- **ICMP** is the go-to for **connectivity** and **path** checks.
- **Routing protocols** build and share network maps to pick paths.
- **NAT/PAT** lets many private devices reach the Internet via one public IP.
