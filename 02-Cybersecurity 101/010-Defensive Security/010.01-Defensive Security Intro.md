# Defensive Security Intro

## **Purpose**

Defensive security focuses on preventing, detecting, and responding to cyber intrusions. It involves continuous monitoring, system hardening, and user education to protect organizational assets.

## **Core Objectives**

1. **Prevent intrusions** – Secure systems and reduce vulnerabilities.
2. **Detect and respond to intrusions** – Identify and contain threats swiftly.

## **Key Defensive Security Activities**

- **User Cybersecurity Awareness:** Train users to recognize and avoid cyber threats.
- **Asset Management:** Document and monitor all systems and devices.
- **Patching & Updates:** Keep all software and hardware protected against known vulnerabilities.
- **Preventive Security Devices:**
    - **Firewalls:** Control inbound and outbound traffic.
    - **Intrusion Prevention Systems (IPS):** Block traffic matching known attack patterns.
- **Logging & Monitoring:** Track network and system activity to detect unauthorized access or anomalies.

## **Areas of Defensive Security**

### **Security Operations Center (SOC)**

A SOC continuously monitors and analyzes an organization’s security posture.

**Main Responsibilities:**

- **Vulnerabilities:** Identify and ensure timely patching or mitigation.
- **Policy Violations:** Detect breaches of internal security policies (e.g., data uploads).
- **Unauthorized Activity:** Identify compromised accounts or abnormal logins.
- **Network Intrusions:** Detect, analyze, and respond to active attacks.

**Additional Focus:**

- **Threat Intelligence** – gathering and analyzing information on adversaries.

### **Threat Intelligence**

**Definition:** Collecting, processing, and analyzing data to understand current and potential adversaries.

**Goals:**

- Identify threat actors, their **tactics, techniques, and procedures (TTPs)**.
- Enable **threat-informed defense** — proactive preparation based on adversary behavior.
- Produce **actionable intelligence**: recommendations and defensive strategies.

**Data Sources:**

- **Internal:** Network logs, system alerts.
- **External:** Forums, open-source data, security feeds.

**Outcome:** Predict and mitigate attacks through informed defensive measures.

### **Digital Forensics and Incident Response (DFIR)**

**Purpose:** Investigate digital evidence to identify the source, method, and impact of an attack.

**Key Evidence Areas:**

- **File System:** Analyze disk images for program and file traces.
- **System Memory:** Capture volatile data to detect in-memory malware.
- **System Logs:** Review activity logs for attacker actions and remnants.
- **Network Logs:** Examine traffic patterns for intrusion evidence.

### **Incident Response (IR)**

**Goal:** Minimize damage, contain threats, and restore systems quickly.

**Four Phases:**

1. **Preparation:** Build trained teams, tools, and response plans.
2. **Detection & Analysis:** Identify and assess the incident’s scope and severity.
3. **Containment, Eradication & Recovery:** Stop the spread, remove threats, and restore systems.
4. **Post-Incident Activity:** Report findings, share lessons, and strengthen defenses.

### **Malware Analysis**

**Objective:** Understand malicious software behavior to develop defenses.

**Common Malware Types:**

- **Virus:** Self-replicating code that damages files.
- **Trojan Horse:** Malicious software disguised as legitimate.
- **Ransomware:** Encrypts files and demands payment for decryption.

**Analysis Methods:**

1. **Static Analysis:** Examine code without execution (requires assembly knowledge).
2. **Dynamic Analysis:** Run malware in a controlled (sandboxed) environment to observe behavior.

## **Overall Outcome**

Defensive security integrates **prevention**, **detection**, and **response** strategies through human expertise, monitoring systems, and intelligence-driven analysis to build resilient cyber defenses.
