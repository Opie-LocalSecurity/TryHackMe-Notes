# SOC Fundamentals

## **Overview of SOC (Security Operations Center)**

- A **SOC** is a dedicated, centralized facility where a specialized security team **monitors**, **detects**, and **responds** to cyber threats 24/7.
- Purpose: Protect the organization’s critical data (secrets) from unauthorized access, modification, or loss.
- SOCs address evolving threats that traditional security measures cannot handle alone.

## **Core Objectives**

### **Detection**

- **Vulnerabilities:** Identify weaknesses in systems or software that could be exploited.
- **Unauthorized Activity:** Detect unusual or suspicious user behavior (e.g., logins from odd locations).
- **Policy Violations:** Spot breaches of organizational security policies (e.g., sharing confidential data insecurely).
- **Intrusions:** Detect actual unauthorized access or malware infections.

### **Response**

- Support incident response by minimizing impact, investigating root causes, and restoring systems.

## **SOC Pillars: People, Process, and Technology**

A mature SOC requires balance among all three components.

### **A. People**

Human analysts are essential to filter out false positives and make informed decisions.

**Roles include:**

- **SOC Analyst L1:** First responders; perform initial alert triage.
- **SOC Analyst L2:** Conduct deeper analysis and data correlation.
- **SOC Analyst L3:** Handle high-severity incidents, perform proactive threat hunting.
- **Security Engineer:** Deploy and configure security tools.
- **Detection Engineer:** Create and maintain detection logic/rules.
- **SOC Manager:** Oversee SOC operations and communicate with leadership (CISO).

### **B. Process**

Defines how SOC members perform their duties efficiently.

**Key Processes:**

- **Alert Triage:**
    - Analyze alerts using the “5 Ws” (What, When, Where, Who, Why).
    - Example: Malware detection on a host → identify origin, user, and cause.
- **Reporting:**
    - Escalate harmful alerts as tickets with full analysis, evidence, and context.
- **Incident Response & Forensics:**
    - Handle critical incidents via containment, eradication, and recovery.
    - Conduct forensic analysis to uncover root causes.

### **C. Technology**

Tools that enable centralized monitoring, detection, and response.

**Key Technologies:**

- **SIEM (Security Information and Event Management):**
    - Collects and correlates logs from multiple sources.
    - Uses detection rules and machine learning for threat identification.
    - Focus: *Detection*.
- **EDR (Endpoint Detection and Response):**
    - Provides real-time endpoint visibility and automated response options.
- **Firewall:**
    - Monitors and filters network traffic between internal and external networks.

**Other Tools:** Antivirus, IDS/IPS, XDR, SOAR, and EPP — chosen based on organizational needs and threat landscape.

## **Key Takeaways**

- A SOC functions as the **nerve center** of an organization’s cybersecurity operations.
- Success relies on:
    - **People** → skilled analysts and engineers.
    - **Processes** → structured workflows for triage, reporting, and response.
    - **Technology** → integrated tools for detection and automation.
- Continuous monitoring and coordinated action across all three pillars are essential for effective defense.
