# Logs Fundamentals

## 🔍 Introduction to Logs

- **Logs** are digital footprints recording all system or application activities—normal or malicious.
- They’re critical for **security investigations**, **forensics**, and **monitoring**—similar to how detectives piece together physical evidence.

### Key Use Cases

| Use Case | Purpose |
| --- | --- |
| **Security Monitoring** | Detect anomalous or suspicious activities in real time. |
| **Incident Investigation & Forensics** | Determine what happened and perform root cause analysis. |
| **Troubleshooting** | Identify and fix system or application errors. |
| **Performance Monitoring** | Measure and analyze system or app performance. |
| **Auditing & Compliance** | Maintain traceable records for accountability and regulation. |

## 🧾 Types of Logs

Logs are categorized based on the information they record, helping investigators focus on relevant data.

| Log Type | Purpose | Examples |
| --- | --- | --- |
| **System Logs** | Record OS-level operations and issues. | Startup/shutdown, driver/hardware errors. |
| **Security Logs** | Record authentication, authorization, and security changes. | Login events, policy changes, account modifications. |
| **Application Logs** | Track application-specific events. | User interactions, updates, crashes. |
| **Audit Logs** | Capture detailed user and system changes for compliance. | Data access, user activity, policy enforcement. |
| **Network Logs** | Track incoming/outgoing network traffic. | Firewall logs, connection attempts. |
| **Access Logs** | Record access to resources. | Web, API, or database access attempts. |

## 🪟 Windows Event Logs Analysis

Windows logs key system activities in separate categories viewable through **Event Viewer**:

- **Application Logs:** App errors, warnings, compatibility issues.
- **System Logs:** Hardware, driver, startup/shutdown, service events.
- **Security Logs:** Authentication, account, and security policy changes.

### Important Event Fields

- **Description:** Detailed info about the event.
- **Log Name:** Category of the event log.
- **Logged:** Timestamp.
- **Event ID:** Unique identifier for event type.

### Key Event IDs

| Event ID | Description |
| --- | --- |
| **4624** | Successful login |
| **4625** | Failed login |
| **4634** | User logged off |
| **4720** | Account created |
| **4724** | Password reset attempt |
| **4722** | Account enabled |
| **4725** | Account disabled |
| **4726** | Account deleted |

**Event Viewer Tools:**

- **Filter Current Log:** Search for specific Event IDs (e.g., `4624` for successful logins).
- View and analyze logs graphically without external tools.

## 🌐 Web Server Access Logs Analysis

Web servers (e.g., Apache) log every HTTP request made to a website.

**Location:** `/var/log/apache2/access.log`

### Common Log Fields

- **IP Address:** Source of request.
- **Timestamp:** When the request occurred.
- **Request:** Includes HTTP method (`GET`, `POST`) and resource (`/index.html`).
- **Status Code:** Server response (`200` = OK, `404` = Not Found, `500` = Error).
- **User-Agent:** Browser/OS info of the requester.

### Useful Linux Commands for Log Analysis

| Command | Purpose | Example |
| --- | --- | --- |
| `cat` | View or concatenate logs. | `cat access.log` |
| `cat file1 file2 > combined.log` | Merge multiple logs. |  |
| `grep` | Search for patterns or IPs. | `grep "192.168.1.1" access.log` |
| `less` | Paginate logs for easier reading and searching. | `less access.log` |

**In `less`:**

- `Spacebar`: Next page
- `b`: Previous page
- `/pattern`: Search for a string
- `n` / `N`: Next/previous match

## 🧠 Key Takeaways

- Logs are central to **security monitoring, troubleshooting, and forensics**.
- **Categorization** of logs simplifies investigation.
- **Windows Event Viewer** and **Linux command-line tools** (`cat`, `grep`, `less`) are essential for manual log analysis.
- Knowing **common Event IDs** and **log formats** greatly speeds up incident response.
