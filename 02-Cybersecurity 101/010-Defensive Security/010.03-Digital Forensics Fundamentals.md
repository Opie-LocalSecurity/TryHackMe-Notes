# Digital Forensics Fundamentals

## **Introduction to Digital Forensics**

- **Digital forensics**: Application of forensic science to investigate crimes involving digital devices.
- **Cyber crime**: Any illegal activity involving computers, networks, or digital devices.
- **Purpose**: Collect, preserve, analyze, and report digital evidence to support legal actions.
- **Example case**: Investigators analyze a suspect’s laptop, hard drive, phone, and USB to uncover digital maps, security documents, photos, and chat records related to a bank robbery.

## **Digital Forensics Methodology (NIST Model)**

NIST defines **four phases** of digital forensics:

1. **Collection**
    - Identify and secure all potential digital evidence (PCs, USBs, phones, etc.).
    - Prevent data tampering and document every collected item.
2. **Examination**
    - Filter and extract relevant data (e.g., media from specific dates, user-specific data).
3. **Analysis**
    - Correlate evidence to reconstruct events and understand the sequence of actions.
4. **Reporting**
    - Prepare detailed reports with findings, methodologies, and recommendations.
    - Include both technical and executive summaries.

## **Types of Digital Forensics**

- **Computer forensics** – Analyzes computers for files, logs, and traces.
- **Mobile forensics** – Extracts data such as calls, messages, and GPS from phones.
- **Network forensics** – Focuses on traffic logs and network events.
- **Database forensics** – Investigates unauthorized data access or alteration.
- **Cloud forensics** – Examines evidence stored in cloud environments.
- **Email forensics** – Traces phishing, spam, and fraudulent communication.

## **Evidence Acquisition**

### **Key Principles**

- **Proper Authorization**
    - Obtain legal approval before collecting data; unauthorized collection is inadmissible in court.
- **Chain of Custody**
    - Maintain a documented trail of evidence handling, including:
        - Description, collector, timestamps, storage location, and access logs.
    - Ensures evidence integrity and accountability.
- **Use of Write Blockers**
    - Prevents any modification of data during acquisition by blocking write operations.
    - Ensures forensic copies are authentic and unaltered.

## **Windows Forensics**

### **Forensic Images**

- **Disk Image (non-volatile)** – Bit-by-bit copy of HDD/SSD (documents, history, media).
- **Memory Image (volatile)** – Captures active RAM data (running processes, connections); must be taken first before power-off.

### **Common Tools**

| Tool | Purpose |
| --- | --- |
| **FTK Imager** | Creates and analyzes disk images (GUI-based). |
| **Autopsy** | Open-source analysis platform (search, recover deleted files, view metadata). |
| **DumpIt** | Command-line tool for memory imaging. |
| **Volatility** | Analyzes memory images across OSes using plugins. |

## **Practical Forensics Example**

### **Document Metadata Analysis**

- Files like PDFs or Word documents contain metadata (creator, creation/modification time).
- **Tool:** `pdfinfo DOCUMENT.pdf`
    - Reveals metadata such as:
        - Creator/Producer (e.g., MS Word)
        - Creation/Modification timestamps
        - File size, version, and properties

### **Photo EXIF Data**

- **EXIF (Exchangeable Image File Format)**: Embedded metadata in images containing:
    - Camera/phone model, date/time, photo settings
    - **GPS coordinates** (location of photo)
- **Tool:** `exiftool IMAGE.jpg`
    - Displays metadata including GPS Position (latitude/longitude)
    - Coordinates can be mapped via Google/Bing Maps to identify the location of capture.

## **Key Takeaways**

- Digital forensics ensures **legal validity, integrity, and accuracy** of digital evidence.
- The **NIST process (Collection → Examination → Analysis → Reporting)** standardizes investigations.
- Proper documentation (**chain of custody**) and technical precautions (**write blockers**) maintain credibility.
- Tools like **FTK Imager, Autopsy, DumpIt, Volatility, pdfinfo,** and **exiftool** are essential for different forensic tasks.
- Metadata and EXIF data often reveal hidden but crucial information (authorship, creation time, geolocation).
