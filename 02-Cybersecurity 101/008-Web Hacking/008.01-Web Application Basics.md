# Web Application Basics

## URL (Uniform Resource Locator)

- **Purpose:** Identifies and locates a resource on the web (page, API, media).
- **Parts & security notes:**
    - **Scheme:** `http` / `https` (TLS-encrypted). Prefer/enforce **HTTPS**.
    - **User:** Optional credentials in URL; **avoid** (security risk).
    - **Host/Domain:** Unique site identifier. Watch for **typosquatting** in phishing.
    - **Port:** Service entry (default **80** HTTP, **443** HTTPS).
    - **Path:** Resource location on server; protect sensitive paths with authz.
    - **Query string `?k=v`:** Client-modifiable; validate to prevent **injection**.
    - **Fragment `#...`:** Client-side anchor; also validate if processed.

## HTTP Messages (Client тЖФ Server)

- **Types:** **Request** (client тЖТ server) and **Response** (server тЖТ client).
- **Structure (both):** **Start line** тЖТ **Headers** тЖТ **Empty line** тЖТ **Body** (optional).
    - **Start line:** Declares intent (method/target/version for requests; version/status for responses).
    - **Headers:** KeyтАУvalue metadata controlling behavior, formats, caching, security.
    - **Body:** Payload (form data, JSON, HTML, files, etc.).

**Why it matters:** Core to web app function, troubleshooting, performance, and security hardening.

## HTTP Request тАФ Request Line & Methods

- **Format:** `METHOD /path HTTP/version`
- **Common methods & security:**
    - **GET:** Retrieve; no side effects. DonтАЩt expose secrets (URLs/logs/cacheable).
    - **POST:** Create/submit data. **Validate & sanitize** (SQLi/XSS).
    - **PUT:** Replace/update. Enforce **authorization**.
    - **DELETE:** Remove. Enforce **authorization**.
    - **PATCH:** Partial update; validate patch operations.
    - **HEAD:** Headers only (no body) for checks.
    - **OPTIONS:** Advertises allowed methods (CORS, discovery).
    - **TRACE:** Echo for debugging; usually **disable**.
    - **CONNECT:** Tunnel for TLS; used by proxies.
- **URL Path security:** Validate, normalize/sanitize, and protect sensitive identifiers.
- **HTTP Versions:**
    - **0.9** (GET only) тЖТ **1.0** (headers) тЖТ **1.1** (persistent, chunking) тЖТ **2** (multiplexing, HPACK) тЖТ **3** (QUIC).
    - **Upgrade to 2/3** for performance & improved transport security where possible.

## HTTP Request тАФ Headers & Body

- **Common request headers:**
    - `Host` (target virtual host)
    - `User-Agent` (client info)
    - `Referer` *(sic)* (source URL)
    - `Cookie` (state from server)
    - `Content-Type` (payload media type)
- **Body formats (typical for POST/PUT/PATCH):**
    - **URL-Encoded** `application/x-www-form-urlencoded`: `k=v&k2=v2` (percent-encoding).
    - **Form-Data** `multipart/form-data`: file uploads; boundary-separated parts.
    - **JSON** `application/json`: `{ "k": "v" }`.
    - **XML** `application/xml`: `<k>v</k>`.
- **Controls:** Validate content type, size, and schema; apply authn/authz.

## HTTP Response тАФ Status Line, Codes, Headers, Body

- **Status line:** `HTTP/version status-code reason-phrase`.
- **Code classes:**
    - **1xx** Informational
    - **2xx** Success
    - **3xx** Redirection
    - **4xx** Client error
    - **5xx** Server error
- **Common codes:** `100 Continue`, `200 OK`, `301 Moved Permanently`, `404 Not Found`, `500 Internal Server Error`.
- **Required/typical response headers:**
    - `Date` (response timestamp)
    - `Content-Type` (media type + charset)
    - `Server` (software; often obfuscate/remove)
- **Other important headers:**
    - `Set-Cookie`: use `HttpOnly`, `Secure` (and `SameSite`) for safety.
    - `Cache-Control`: cache policy (`no-store`, `max-age`, etc.).
    - `Location`: redirect target (validate to prevent **open redirects**).
- **Response body:** Returned representation (HTML/JSON/media). **Escape/sanitize** user content to prevent **XSS**.

## Security Headers (Key Hardening Controls)

- **Content-Security-Policy (CSP):** Whitelists content sources; e.g.
    
    `Content-Security-Policy: default-src 'self'; script-src 'self' https://cdn.example.com; style-src 'self'`
    
- **Strict-Transport-Security (HSTS):** Force HTTPS; e.g.
    
    `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload`
    
- **X-Content-Type-Options:** Disable MIME sniffing;
    
    `X-Content-Type-Options: nosniff`
    
- **Referrer-Policy:** Control referrer leakage; e.g.
    
    `no-referrer` | `same-origin` | `strict-origin` | `strict-origin-when-cross-origin`
    

**Tooling tip:** Use **securityheaders.io** to audit deployed headers and identify gaps.

## Quick Security Checklist

- Enforce **HTTPS** (+ **HSTS**).
- Validate/sanitize **paths**, **queries**, **bodies**; use proper **Content-Type**.
- Apply **authn/authz** to state-changing methods (PUT/PATCH/DELETE).
- Minimize sensitive data in URLs; use secure cookies (`HttpOnly`, `Secure`, `SameSite`).
- Set **CSP**, **X-Content-Type-Options**, **Referrer-Policy**, and robust **Cache-Control**.
- Disable unnecessary methods (**TRACE**, unused verbs).
