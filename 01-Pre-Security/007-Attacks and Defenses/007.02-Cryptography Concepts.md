# Cryptography Concepts

Cryptography is the practical application of mathematical algorithms to satisfy the **CIA Triad**, specifically protecting **Confidentiality** (secrecy) and **Integrity** (detecting tampering).

## 1. Core Terminology

- **Plaintext:** The original, readable message.
- **Ciphertext:** The encrypted, unreadable "gibberish."
- **Algorithm:** The public "recipe" or set of mathematical steps used to transform data.
- **Key:** The secret parameter required by the algorithm to encrypt or decrypt.

## 2. Symmetric Encryption

In symmetric encryption, the **same key** is used for both encryption and decryption.

- **Methodology:** Plaintext + Algorithm + Secret Key → Ciphertext.
- **The Caesar Cipher:** A basic substitution cipher that shifts letters by a fixed number (the key). While educational, it is insecure due to its small key space (25 possible shifts).
- **Strengths:** Extremely fast and efficient; ideal for bulk data (hard drives, large files).
- **The Key Distribution Problem:** The "Achilles' heel" of this method. If two parties haven't met, they cannot securely exchange the secret key over an untrusted network without it being intercepted.

## 3. Asymmetric Encryption (Public Key Cryptography)

Asymmetric encryption uses a **mathematically linked key pair** to solve the distribution problem.

- **Public Key:** Shared openly; used by others to **encrypt** messages for you.
- **Private Key:** Kept strictly secret; used only by the owner to **decrypt** messages.
- **The Mailbox Analogy:** The public key is the mail slot (anyone can drop mail in), and the private key is the door key (only the owner can take mail out).
- **Strengths:** Eliminates the need to share a secret key beforehand.
- **Weaknesses:** Computationally intensive and significantly slower than symmetric encryption.

## 4. Practical Implementation: The Hybrid Approach

Modern secure communication (like **HTTPS**) does not choose one over the other; it combines them for maximum efficiency and security.

### How HTTPS Works:

1. **Asymmetric Phase:** The browser uses the website's **Public Key** to securely negotiate a temporary "Session Key."
2. **Symmetric Phase:** Both parties switch to **Symmetric Encryption** using that Session Key for the remainder of the visit because it is faster for data transfer.

### Digital Certificates & Trust:

To prevent "impersonation" attacks (where an attacker provides a fake public key), **Certificate Authorities (CAs)** issue digital certificates.

- **Function:** Validates that a public key actually belongs to a specific domain (e.g., `google.com`).
- **Verification:** Browsers check the CA's signature and the expiration date before displaying the "padlock" icon.

## Comparison Table

| **Feature** | **Symmetric Encryption** | **Asymmetric Encryption** |
| --- | --- | --- |
| **Key Count** | One (Shared) | Two (Public & Private) |
| **Speed** | High (Fast) | Low (Slower) |
| **Primary Use** | Bulk data, file encryption | Key exchange, digital signatures |
| **Example** | AES | RSA, ECC |
