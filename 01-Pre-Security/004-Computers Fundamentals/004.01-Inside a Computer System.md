# Inside a Computer System

### Phase 1: Power and Initialization

- **Power Signal:** Pressing the button signals the **Power Supply Unit (PSU)** to distribute electricity to the motherboard and components.
- **Firmware Activation:** The system initializes the **UEFI** (Unified Extensible Firmware Interface) or the legacy **BIOS** (Basic Input/Output System). This low-level software acts as the intermediary between the hardware and the OS.

### Phase 2: Hardware Validation

- **POST (Power-On Self Test):** The UEFI executes a diagnostic routine to ensure critical hardware (CPU, RAM, Storage, etc.) is present, configured, and functional.
- **Error Handling:** If the POST fails, the system emits "beep codes" or visual error messages to identify the faulty component.

### Phase 3: The Boot Hand-off

- **Boot Device Selection:** The UEFI consults a prioritized list (the **Boot Order**) to find where the Operating System is stored (e.g., SSD, HDD, or USB).
- **Bootloader Initiation:** Once the device is found, the UEFI starts the **Bootloader**. This small program is responsible for:
    1. Locating the Operating System kernel.
    2. Loading the OS into the **RAM** (Random Access Memory).
    3. Transferring hardware control from the UEFI to the Operating System.
