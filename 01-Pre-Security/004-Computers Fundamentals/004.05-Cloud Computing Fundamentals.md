# Cloud Computing Fundamentals

Cloud computing is the delivery of computing services—including servers, storage, databases, and software—over the internet. It transitioned from traditional physical hardware to a virtualized, scalable model to reduce costs and increase efficiency.

## Key Benefits & Characteristics

- **Scalability:** Rapidly adjust resources (CPU, RAM, Storage) up or down based on demand.
- **Elasticity/On-Demand:** Instant provisioning of resources without waiting for physical hardware.
- **Cost Efficiency:** "Pay-as-you-go" model eliminates high upfront capital expenditures (CapEx).
- **High Availability:** Redundant infrastructure ensures applications remain online during local failures.
- **Global Reach:** Deploy applications in multiple geographical regions to reduce latency for end-users.

## Cloud Models

### 1. Deployment Models

- **Public Cloud:** Resources shared across multiple organizations (e.g., AWS, Azure). Best for scalability and cost-savings.
- **Private Cloud:** Dedicated infrastructure for a single organization. Best for high security and strict compliance.
- **Hybrid Cloud:** A mix of public and private clouds, allowing data and apps to move between them for flexibility.

### 2. Service Models (The Responsibility Spectrum)

- **Infrastructure as a Service (IaaS):** Provides virtualized hardware (servers, networking). You manage the OS and apps. (Example: **AWS EC2**)
- **Platform as a Service (PaaS):** Provides a framework for developers. You manage the code; the provider manages the OS and hardware.
- **Software as a Service (SaaS):** Fully managed software accessed via a browser. (Examples: **Gmail, Zoom**)

## Core Technical Concepts (AWS Focus)

- **Region:** A specific geographical area (e.g., US-East-1) containing data centers. Choosing a region affects latency and cost.
- **EC2 (Elastic Compute Cloud):** A virtual server (Instance) in the cloud.
- **Instance Type:** Defines the hardware capacity.
    - *t3.micro:* Low power, lower cost (General purpose/testing).
    - *m5.large:* Higher CPU/RAM, higher cost (Production/intensive tasks).
- **Lifecycle Management:** Instances can be **Running** (incurring costs) or **Stopped** (minimizing costs while retaining configuration).

## Practical Outcome: Cost Optimization

The primary takeaway from cloud deployment is the ability to perform **Billing Analysis**. By monitoring usage and stopping non-essential instances (like "study machines" during off-hours), organizations can significantly reduce operational expenses (OpEx).
