# Client-Server Basics

## 1. Core Architecture: Client, Server, and Service

The **Client-Server model** is the foundation of network communication.

- **Client:** The entity that **initiates** a request (e.g., a web browser).
- **Server:** The system that provides the resource or **service** (e.g., a web server hosting a site).
- **Service:** A specific function or application running on a server.
- **Request & Response:** Communication is a two-way exchange. A malformed request or missing resource results in an **error response**.

## 2. Communication Enablers: Protocols, Ports, and DNS

For a client to successfully reach and talk to a server, three technical elements must align:

- **Protocol:** A set of rules defining the "language" of communication. It specifies command types (e.g., `GET`), request structure, syntax, and how to handle errors.
- **Port:** A virtual "door" used to identify a specific service on a server. A single server can host multiple services (Web, Email, FTP) by assigning each to a unique port.
- **DNS (Domain Name Service):** A system that translates human-readable domain names (e.g., `google.com`) into machine-readable **IP Addresses** (e.g., `142.250.190.46`).

## 3. Web Communications: HTTP(S)

**Hypertext Transfer Protocol (Secure)** is the primary protocol for the web.

- **Statelessness:** By default, HTTP does not remember previous interactions. Each request is independent.
- **Statefulness:** To provide continuity (like staying logged in), applications use **Session Identifiers** stored in **cookies** or **tokens**.

### HTTP Methods (Commands)

There are 9 core methods used to tell the server what action to perform. Common methods include:

- **GET:** Retrieve data from a server.
- **POST:** Send data to a server (e.g., a form submission).
- **PUT/PATCH:** Update existing data.
- **DELETE:** Remove data.

## 4. Anatomy of an HTTP GET Request

When you navigate to a URL, the browser automatically constructs a GET request.

### Key Request Fields

- **Scheme:** The protocol used (`http` or `https`).
- **Host:** The target domain name.
- **Filename:** The specific resource requested (e.g., `/` often defaults to `index.html`).
- **Address:** The IP address of the host (e.g., `127.0.0.1` for a local host).

### The Response

The server's reply consists of two distinct sections:

1. **Response Header:** Metadata about the transmission, including the **Status Code** (e.g., `200 OK` for success).
2. **Response Body:** The actual content requested, often formatted in **HTML**
