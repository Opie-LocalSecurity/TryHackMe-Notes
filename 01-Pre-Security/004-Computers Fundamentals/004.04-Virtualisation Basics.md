# Virtualisation Basics

Virtualization is the technology that allows multiple simulated environments (Virtual Machines) to run on a single physical hardware unit. It moves IT away from the "one server, one application" model toward a more efficient, shared-resource architecture.

## 1. Key Concepts & Drivers

- **The Problem:** Physical servers often suffer from **low utilization** (5–20%), high overhead costs (power/cooling), and slow deployment times.
- **The Solution:** Using a software layer to "slice" hardware into independent segments.
- **The Analogy:** A physical server is a **building**; a Hypervisor is the **building manager**; Virtual Machines are **apartments**; and Containers are **rooms** within those apartments.

## 2. Core Methodologies

### **Hypervisors (The Management Layer)**

The software that creates, isolates, and manages resources (CPU, RAM, Disk) for VMs.

- **Type 1 (Bare Metal):** Runs directly on hardware. Optimized for speed and efficiency. Used for **Production Servers, Data Centers, and Databases.**
- **Type 2 (Hosted):** Runs on top of an existing OS (e.g., Windows/macOS). Used for **Software Testing, Malware Analysis, and Learning Labs.**

### **Virtual Machines (VMs)**

- **Nature:** A full "virtual computer" with its own dedicated Operating System.
- **Isolation:** High. If one VM crashes or is infected by malware, others remain unaffected.
- **Tools:** Oracle VirtualBox, VMware Workstation.

### **Containers (The Lightweight Alternative)**

- **Nature:** Packages an application and its dependencies, but **shares the host OS kernel**.
- **Efficiency:** Starts instantly and uses fewer resources than VMs because they don't boot a full OS.
- **Constraint:** Must match the host OS type (e.g., Linux containers need a Linux host).
- **Tools:** Docker.

## 4. Operational Management (Practical Skills)

To maintain a virtualized environment, administrators use a **Virtualization Manager** to monitor three key areas:

- **Summary:** High-level health of the entire environment.
- **VM Management:** Monitoring states (Running, Error, Stopped) and performing lifecycle actions (Restart, Create, Delete).
- **Host Management:** Monitoring physical hardware capacity.
    - *Critical Metric:* Identifying "hot" hosts (operating near 100% capacity) versus underutilized or disconnected hosts.

## 5. Outcomes & Benefits

- **Resource Efficiency:** Maximizes hardware usage by hosting multiple services on one box.
- **Agility:** New servers (VMs) can be created in minutes via software forms rather than weeks of hardware procurement.
- **Security:** Provides isolated "sandboxes" for testing dangerous files without risking the host machine.
