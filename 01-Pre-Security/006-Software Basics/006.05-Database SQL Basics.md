# Database SQL Basics

## 1. Core Concepts: Databases vs. Manual Tracking

- **The Problem:** Manual systems (like paper notebooks) become inefficient as data grows, making searching, counting, and sorting slow and error-prone.
- **The Solution:** A **Database** acts as an organized digital storage system designed for high-speed data manipulation and persistence.

## 2. Data Hierarchy and Structure

Information is organized into **Tables**, which function similarly to spreadsheets:

- **Columns (Attributes):** The vertical categories (titles) that define the *type* of data being stored (e.g., `drink_name`, `price`).
- **Rows (Records):** The horizontal entries representing a *single complete unit* of data (e.g., one specific customer order).
- **Scalability:** Adding or removing data involves adding or deleting specific rows without altering the overall table structure.

## 3. SQL Methodology: Querying Data

**SQL** is the standard language used to communicate with databases. A **Query** is a request for specific information from one or more tables that *does not* modify the original data.

### Key SQL Clauses and Syntax:

| **Clause** | **Purpose** | **Example** |
| --- | --- | --- |
| **SELECT** | Specifies the columns to display (use `*` for all). | `SELECT drink, price` |
| **FROM** | Identifies the table containing the data. | `FROM Orders;` |
| **WHERE** | Filters results based on a specific condition. | `WHERE drink = 'Coffee'` |
| **ORDER BY** | Sorts the output by a specific column (defaults to ascending). | `ORDER BY price` |
| **DESC** | Reverses the sort order (descending: highest to lowest). | `ORDER BY price DESC` |

## 4. Outcomes and Skills Developed

- **Data Extraction:** Learned how to retrieve all data (`SELECT *`) or specific attributes from a table.
- **Data Refinement:** Applied filtering techniques (`WHERE`) to isolate records of interest (e.g., only "Coffee" orders).
- **Data Organization:** Utilized sorting mechanisms (`ORDER BY`) to present information in a logical sequence.
- **Complex Querying:** Combined filtering and sorting into a single, cohesive command for precise data analysis.
