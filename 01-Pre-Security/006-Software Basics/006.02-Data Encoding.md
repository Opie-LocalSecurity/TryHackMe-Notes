# Data Encoding

## 1. ASCII (American Standard Code for Information Interchange)

ASCII is the foundational standard for representing text as numerical data.

- **Core Concept:** A 7-bit encoding scheme providing 128 unique code points ($2^7$).
- **Scope:** Includes English uppercase/lowercase letters, digits (0-9), punctuation, and control characters (e.g., `DEL`, `New Line`).
- **Methodology:** Maps specific binary patterns to characters. For example, `A` is always `01000001` (Decimal 65 / Hex 41).
- **Key Characteristic:** Characters are sequential (e.g., if 'a' is `0x61`, 'b' is `0x62`), making programmatic sorting and calculation simple.

## 2. Extended ASCII & ISO-8859 Series

As computing expanded globally, the original 7-bit ASCII was insufficient for non-English characters.

- **The 8th Bit:** By using 8 bits (1 byte), the range expanded to 256 characters ($2^8$).
- **ISO-8859 Standards:** A series of regional standards where the first 128 characters match ASCII, but the remaining 128 vary by region.
    - **ISO-8859-1 (Latin-1):** Western European (German, French, Spanish).
    - **ISO-8859-2 (Latin-2):** Central/Eastern European (Polish, Czech, Romanian).
- **The "Chaos" Outcome:** Incompatibility. A file saved in Latin-1 might display incorrect "garbage" characters if opened with a Latin-2 reader because the same numerical code maps to different symbols in different regions.

## 3. Unicode

Unicode was developed to solve regional fragmentation by creating a single, universal repertoire for every character in human history.

- **Concept:** Assigns a unique **Code Point** (written as `U+XXXX`) to every character, including emojis, ancient scripts, and complex Asian logograms (Kanji/Hanzi).
- **Scale:** The latest version (v17.0) defines nearly 157,000 characters.

## 4. Encoding Methods (UTF)

While Unicode defines the "map," **UTF (Unicode Transformation Format)** defines how those numbers are actually stored in bits.

| **Format** | **Bit Width** | **Characteristics** |
| --- | --- | --- |
| **UTF-8** | **Variable** (1–4 bytes) | **Most common.** Efficient; backward compatible with ASCII (uses 1 byte for English). |
| **UTF-16** | **Variable** (2 or 4 bytes) | Common in Windows/Java environments. Uses at least 2 bytes even for simple text. |
| **UTF-32** | **Fixed** (4 bytes) | Simplest for indexing but very wasteful (uses 4 bytes for every single character). |

## Key Takeaways for Review

- **Computers only see bits:** Encoding is the "dictionary" that tells the computer which bits represent which visual glyphs.
- **Hexadecimal Preference:** Engineers use Hex (e.g., `0x0A` for a new line) instead of binary because it is more human-readable while remaining technically precise.
- **Compatibility:** Modern development defaults to **UTF-8** to ensure a file looks the same regardless of the user's language or region.
