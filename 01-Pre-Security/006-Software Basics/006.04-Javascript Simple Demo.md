# Javascript: Simple Demo

## 1. Environment & Setup

- **Execution Runtimes:** JavaScript can be executed in a **Web Browser** (via Developer Tools/Console) or via **Node.js** (command line).
- **Modules:** To handle command-line interaction in Node.js, specific modules are required:
    - `node:readline/promises`: Enables asynchronous user prompting.
    - `node:process`: Provides access to `stdin` (keyboard) and `stdout` (screen).
- **Clean-up:** The `readline` interface (e.g., `rl.close()`) must be explicitly closed to terminate the program properly and free resources.

## 2. Key JavaScript Concepts & Variables

### Variables vs. Constants

- **`let` (Variables):** Used for values that change during execution (e.g., `tries`, `guess`).
- **`const` (Constants):** Used for values that remain fixed once assigned (e.g., the `secret` number).

### Math Operations

To generate a random integer within a specific range (1–20):

- `Math.random()`: Generates a decimal between 0 and 0.999...
- `Math.floor()`: Rounds down to the nearest whole number.
- **Formula:** `Math.floor(Math.random() * (20)) + 1`

## 3. Data Flow & Logic

### Input/Output (I/O)

- **Display:** `console.log()` outputs text to the terminal.
- **Prompting:** `await rl.question()` pauses execution until the user provides a response.
- **Parsing:** `parseInt(text, 10)` converts a string input into a base-10 integer for mathematical comparison.

### Control Flow

- **Conditional Statements:** Uses `if`, `else if`, and `else` to evaluate the user’s guess:
    - **Range Check:** Uses the `||` (OR) operator (e.g., `guess < 1 || guess > 20`).
    - **Comparison:** Evaluates if the guess is higher or lower than the `secret`.
- **Logical Operators:** `!==` (Not Equal) and `||` (Logical OR).

### Iteration (Loops)

- **`while` Loop:** Repeats a block of code continuously as long as a condition is true (e.g., `while (guess !== secret)`). This allows the user multiple attempts until the correct number is found.

## 4. Error Handling & Structure

- **`try...finally` Block:** Ensures that critical clean-up code (`rl.close()`) executes even if an error occurs within the main logic.
