# Python: Simple Demo

## 1. Core Concepts & Variables

The program relies on three primary variables to manage the state of the game:

- **`secret`**: An integer generated randomly using `random.randint(1, 20)`.
- **`guess`**: Stores the user's current attempt (initialized to `0` to ensure it doesn't accidentally match the secret at start).
- **`tries`**: A counter that increments by 1 with every user input to track performance.

## 2. Key Python Tools & Methods

The implementation utilizes several built-in functions to handle data and interaction:

- **`random.randint(a, b)`**: Generates a random integer where $a \le \text{result} \le b$.
- **`input()`**: Captures user input as a **string** (text).
- **`int()`**: A type-casting function used to convert the string input into an integer for mathematical comparison.
- **`print()`**: Outputs feedback and instructions to the console.

## 3. Logic & Methodology

The program's intelligence is built through conditional logic and iteration.

### Conditional Branching (`if/elif/else`)

To provide feedback, the program evaluates the `guess` against the `secret` using comparison operators:

1. **Out of Bounds**: Checks if `guess < 1 or guess > 20`.
2. **Too Low**: Checks if `guess < secret`.
3. **Too High**: Checks if `guess > secret`.
4. **Correct**: The `else` block triggers when no other conditions are met (meaning `guess == secret`).

### Iteration (`while` loop)

To allow multiple attempts, the code uses a `while` loop.

- **Condition**: `while guess != secret:`
- **Outcome**: The code block (input, incrementing tries, and feedback) repeats indefinitely until the user's guess matches the secret number, at which point the loop condition becomes `False` and the program terminates.

## 4. Final Program Structure

The evolution of the code resulted in the following workflow:

1. **Initialize**: Import `random`, set the `secret`, and reset `tries`.
2. **Loop**: While the guess is incorrect, prompt for a new number.
3. **Process**: Convert input to integer and increment the counter.
4. **Feedback**: Inform the user if the guess is out of range, too high, or too low.
5. **Termination**: If the guess matches the secret, exit the loop and display the final count of tries.
