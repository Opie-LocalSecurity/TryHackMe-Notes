# Operating System Security

## 1. The Role of the Operating System

The **Operating System (OS)** serves as the essential intermediary layer between computer **hardware** (CPU, RAM, Storage) and **user applications** (Browsers, Messaging apps). It manages hardware resources and enforces rules for how programs interact with the physical machine.

- **Desktop/Laptop OS:** Windows, macOS, Linux.
- **Mobile OS:** iOS, Android.
- **Server OS:** Windows Server, IBM AIX, Oracle Solaris, Linux.

## 2. Core Security Principles (CIA Triad)

OS security aims to protect sensitive data (passwords, private files, banking info) by upholding three pillars:

- **Confidentiality:** Ensuring data is accessible only to authorized users.
- **Integrity:** Guaranteeing that data has not been tampered with or altered.
- **Availability:** Ensuring systems and data are accessible when needed.

## 3. Common OS Vulnerabilities

Attackers typically target three primary weaknesses to compromise a system:

### A. Authentication & Weak Passwords

Authentication verifies identity via three factors: **Knowledge** (passwords), **Inherence** (biometrics), or **Possession** (security tokens).

- **The Threat:** Users often use "predictable" passwords (e.g., `123456`, `password`, or keyboard patterns like `qwerty`).
- **Best Practice:** Use unique, complex passwords for every account.

### B. Weak File Permissions

- **Principle of Least Privilege:** Users should only have the minimum level of access necessary to perform their jobs.
- **The Threat:** Incorrect permissions allow unauthorized users to read (violating confidentiality) or modify (violating integrity) sensitive files.

### C. Malicious Programs (Malware)

- **Trojan Horses:** Disguised as legitimate software to provide attackers remote access.
- **Ransomware:** Encrypts user files to block access (violating availability) until a ransom is paid for the decryption key.

## 4. Practical Methodology & Linux Commands

In a technical environment (specifically Linux), security testing often involves identifying user accounts and attempting to gain access through credential harvesting or privilege escalation (moving from a standard user to **Root/Administrator**).

| **Command** | **Function** |
| --- | --- |
| `ssh USER@IP` | Securely connect to a remote system. |
| `whoami` | Displays the current logged-in username. |
| `ls` | Lists files in the current directory. |
| `cat FILENAME` | Displays the contents of a file. |
| `history` | Shows a list of previously executed commands. |
| `su - USER` | Switch to another user account (requires password). |
