# Operating Systems: Introduction

This summary covers the foundational concepts of Operating Systems (OS), focusing on their role as the "Invisible Manager" of a computer's hardware and software.

## 1. Core Definition & Role

An **Operating System (OS)** is the intermediary software that coordinates interactions between the user, applications, and physical hardware.

- **The Airport Analogy:** * **Hardware:** Infrastructure (runways, planes).
    - **Applications:** Users/Airlines (requesting services).
    - **OS:** Air Traffic Control (scheduling, safety, and resource allocation).
- **Purpose:** To prevent resource conflicts that would occur if every app tried to control the CPU or memory simultaneously.

## 2. System Privilege Layers

To ensure stability and security, modern operating systems separate operations into two distinct "spaces":

| **Layer** | **Access Level** | **Function** |
| --- | --- | --- |
| **Kernel Space** | Privileged | The core "Control Tower." Has unrestricted access to hardware (CPU, RAM, Storage). |
| **User Space** | Restricted | Where standard apps run. They cannot touch hardware directly; they must send **System Calls** to the Kernel to perform tasks. |

## 3. Key OS Responsibilities

The OS performs five primary duties to maintain a functional environment:

- **Process Management:** Scheduling and prioritizing CPU time for multiple apps.
- **Memory Management:** Allocating RAM and using **Virtual Memory** to prevent crashes.
- **File System Management:** Organizing data, managing permissions, and metadata.
- **User Management:** Handling authentication (biometrics/passwords) and account isolation.
- **Device Management:** Using **drivers** to provide a hardware abstraction layer (e.g., "plug and play").

## 4. Interfaces: GUI vs. CLI

Users interact with the OS through two primary methods:

1. **Graphical User Interface (GUI):** Visual-based (icons, windows, menus). Best for ease of use and daily tasks.
2. **Command-Line Interface (CLI):** Text-based syntax. Best for precision, speed, and advanced administrative tasks.

## 5. The OS Landscape & Families

Operating systems are specialized based on their environment (stability vs. power efficiency vs. footprint).

### OS Categories

- **Desktop:** User-focused, rich GUI (Windows, macOS, Linux).
- **Server:** High uptime, often "headless" (no GUI), remote access (Linux, Windows Server).
- **Mobile:** Touch-based, power-efficient, app sandboxing (Android, iOS).
- **Embedded/IoT:** Tiny footprint for specific tasks like routers or cars (FreeRTOS, Embedded Linux).
- **Virtual/Cloud:** Lightweight, scalable instances for data centers (Amazon Linux, Alpine).

### Major Families

- **Windows:** Dominant in desktop and corporate environments.
- **Linux:** Open-source family; dominates servers, cloud, and embedded systems.
- **Apple (macOS/iOS):** Polished, closed ecosystem known for hardware-software integration.
- **Unix:** Legacy/Enterprise systems used in high-stakes environments like finance.
