# Linux CLI Basics

## 1. Terminal Fundamentals

The terminal is a text-based interface used for direct system control. It is preferred by professionals for its speed, granular control, and compatibility with security tools that lack graphical interfaces.

## 2. Navigation & File Exploration

Moving through the Linux filesystem requires understanding your current location and the contents around you.

| **Command** | **Action** | **Key Flags / Arguments** |
| --- | --- | --- |
| `pwd` | **Print Working Directory**: Shows your current location. | N/A |
| `ls` | **List**: Shows files and folders in the current directory. | `-l` (long format/details), `-a` (show hidden files starting with `.`) |
| `cd` | **Change Directory**: Moves you to a different folder. | `..` (move up one level), `~` (go to home directory) |
| `find` | **Locate Files**: Searches the system for specific files. | `find <path> -name <filename>` |
| `cat` | **Concatenate**: Reads and displays the content of a file. | `cat <filename>` |

## 3. System Reconnaissance

Identifying system specifications is a critical first step in environment auditing or troubleshooting.

- **Identity**: Use `whoami` to confirm the current logged-in user account.
- **System Info**: Use `uname -a` to view the kernel version, hostname, and hardware architecture (e.g., `x86_64`).
- **Storage**: Use `df -h` to check disk space. The `h` flag converts bytes into **human-readable** formats (GB, MB).
- **Distribution Details**: Read the `/etc/os-release` file using `cat` to find the specific Linux distribution name (e.g., Ubuntu 24.04) and version codename.

## 4. Key Methodologies

1. **Context Awareness**: Always run `pwd` and `whoami` when starting a session to establish your environment and permissions.
2. **Verbose Listing**: Use `ls -al` regularly to ensure you aren't missing hidden configuration files or restricted permissions.
3. **Targeted Searching**: Instead of manual browsing, use `find` with the specific directory start point (like `/` or `~`) to save time.
