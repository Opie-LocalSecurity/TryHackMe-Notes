# Volatility

This summary provides a technical overview of **Volatility 3**, the industry-standard open-source framework for volatile memory (RAM) forensics.

## 1. Framework Overview

Volatility is a Python-based tool used by Blue Teams and Malware Analysts to extract digital artifacts from RAM dumps. It operates **independently of the target system**, ensuring data integrity and preventing anti-forensic measures from triggering on the live host.

- **Language:** Python 3 (Volatility 3 is the current standard).
- **Architecture:** Modular "Plug-and-Play" system using specific plugins for different OS structures.
- **Compatibility:** Windows, Linux, and macOS.
- **Key Change:** Volatility 3 has deprecated "Profiles." It now automatically identifies the OS and build, though you must prefix plugins with the OS (e.g., `windows.pslist`).

## 2. Memory Extraction

Before analysis, memory must be "dumped" into a file (usually `.raw`, `.vmem`, or `.mem`).

| Source Type               | Common Tools / File Extensions               |
| ------------------------- | -------------------------------------------- |
| **Bare-Metal (Physical)** | FTK Imager, Redline, DumpIt.exe, win64dd.exe |
| **VMWare**                | `.vmem`                                      |
| **Hyper-V**               | `.bin`                                       |
| **VirtualBox**            | `.sav` (Note: only partial memory)           |

## 3. Core Analysis Methodologies

The analysis workflow typically moves from high-level system information to specific process and network triaging.

### System Identification

- **`windows.info` / `linux.info`:** Replaces the old `imageinfo`. Provides metadata about the host's runtime state.

### Process Triage

- **`pslist`:** Displays processes via the OS doubly-linked list. Visible to the OS, but easily hidden by rootkits.
- **`psscan`:** Scans for `_EPROCESS` data structures. Can find hidden or unlinked processes that `pslist` misses.
- **`pstree`:** Visualizes the parent-child relationship of processes to identify suspicious execution chains (e.g., `notepad.exe` spawning `cmd.exe`).
- **`dlllist`:** Lists all Dynamic Link Libraries loaded by a specific process.

### Network Analysis

- **`netstat`:** Identifies active network connections.
- _Note:_ Can be unstable in Volatility 3; analysts often use **bulk_extractor** to pull PCAPs directly from the memory file as a fallback.

## 4. Malware Hunting & Advanced Forensics

Advanced detection focuses on finding code that exists only in memory or manipulates kernel structures.

### Detection Plugins

- **`malfind`:** Locates injected code by scanning for memory segments with **RWE** (Read, Write, Execute) permissions and no associated file on disk.
- **`yarascan`:** Runs custom YARA signatures against the memory dump to find specific malware strings or byte patterns.

### Kernel & Evasion Detection

- **Hooking (`ssdt`):** Scans the **System Service Descriptor Table**. Rootkits often "hook" this table to redirect system functions to malicious code.
- **Driver Analysis:** \* **`modules`:** Lists currently loaded kernel modules.
- **`driverscan`:** A deeper scan that finds drivers even if they have been hidden or unloaded from the standard module list.

## 5. Quick Commands Reference

| Goal                   | Command Syntax                             |
| ---------------------- | ------------------------------------------ |
| **Help/List Plugins**  | `python3 vol.py -h`                        |
| **Get Image Info**     | `python3 vol.py -f <file> windows.info`    |
| **View Process Tree**  | `python3 vol.py -f <file> windows.pstree`  |
| **Find Injected Code** | `python3 vol.py -f <file> windows.malfind` |
| **Check for Hooking**  | `python3 vol.py -f <file> windows.ssdt`    |
