# Sysmon

**System Monitor (Sysmon)** is a Windows system service and device driver that provides granular monitoring of system activity. Unlike standard Windows logs, Sysmon offers deep visibility into process creations, network connections, and file changes, making it a critical tool for identifying advanced persistent threats (APTs) and malware.

## Core Functionality & Architecture

- **Persistence:** Remains resident across reboots once installed.
- **Early Boot:** Starts early in the Windows boot process to capture activity before most user-space applications.
- **Log Storage:** Events are written to: `Applications and Services Logs/Microsoft/Windows/Sysmon/Operational`.
- **Integration:** Ideally used with a **SIEM** (Security Information and Event Management) system for centralized analysis and long-term retention.

## Configuration Methodologies

Sysmon requires an **XML configuration file** to define what activity to log.

- **Exclude vs. Include:** \* **Exclude (Recommended):** Filters out "normal" noise (e.g., `svchost.exe`). This reduces the volume of logs and helps analysts focus on anomalies.
- **Include:** Explicitly logs specific activities (e.g., monitoring a specific sensitive directory).

- **Notable Configs:** \* **SwiftOnSecurity:** High-quality, widely used config focused on identifying common anomalies.
- **ION-Storm:** A proactive fork utilizing more "include" rules for specific threats.

## Essential Event IDs (EIDs) for Threat Hunting

| Event ID  | Name                 | Primary Use Case                                                            | Key XML Tags                 |
| --------- | -------------------- | --------------------------------------------------------------------------- | ---------------------------- |
| **1**     | Process Creation     | Detecting suspicious binaries or typosquatted processes.                    | `CommandLine`, `Image`       |
| **3**     | Network Connection   | Identifying C2 beacons, open ports (e.g., 4444), and remote file transfers. | `Image`, `DestinationPort`   |
| **7**     | Image Loaded         | Detecting DLL Injection or Hijacking (Caution: High system load).           | `ImageLoaded`, `Signed`      |
| **8**     | CreateRemoteThread   | Detecting code injection (e.g., Cobalt Strike, Process Hollowing).          | `SourceImage`, `TargetImage` |
| **11**    | File Created         | Identifying ransomware activity or dropped payloads.                        | `TargetFilename`             |
| **12-14** | Registry Event       | Monitoring persistence via "Run" keys or startup scripts.                   | `TargetObject`               |
| **15**    | FileCreateStreamHash | Detecting malware hiding in **Alternate Data Streams (ADS)**.               | `Contents`, `Hash`           |
| **22**    | DNS Event            | Identifying communication with malicious domains (DGA/C2).                  | `QueryName`                  |

## Tooling & Command Line Usage

### **Installation & Execution**

- **Install:** `Sysmon.exe -i <config_file.xml>`
- **Accept EULA & Run:** `Sysmon.exe -accepteula -i swift.xml`
- **Update Config:** `Sysmon.exe -c <new_config.xml>`

### **PowerShell Analysis (`Get-WinEvent`)**

PowerShell is preferred over the Event Viewer GUI for speed and granular filtering using **XPath** queries.

- **Filter by Port:**
  `Get-WinEvent -Path <LogPath> -FilterXPath '*/System/EventID=3 and */EventData/Data[@Name="DestinationPort"]=4444'`
- **Filter by LSASS Access (Mimikatz Hunt):**
  `Get-WinEvent -Path <LogPath> -FilterXPath '*/System/EventID=10 and */EventData/Data[@Name="TargetImage"]="C:\Windows\system32\lsass.exe"'`

## Threat Hunting Applications

- **Metasploit/C2:** Monitor EID 3 for suspicious ports (4444, 5555, 8080) and EID 1 for suspicious parent-child process relationships.
- **Mimikatz:** Monitor EID 10 (**ProcessAccess**) for any process other than `svchost.exe` accessing `lsass.exe`.
- **Persistence:** Monitor EID 11 (File Creation in `\Startup\`) and EIDs 12-14 (Registry modifications to `CurrentVersion\Run`).
- **Evasion:** Use EID 15 to find files with hidden streams and EID 8 to detect **Process Hollowing** (e.g., `powershell.exe` injecting into `notepad.exe`).
