# Autopsy

This summary provides a technical overview of **Autopsy**, a premier open-source digital forensics platform used for analyzing hard drives, mobile devices, and disk images.

## Core Workflow & Case Management

The forensic process in Autopsy follows a linear progression to ensure data integrity and organized analysis:

- **Case Creation:** Analysts define a **Case Name**, **Base Directory** (storage for case metadata), and **Case Type** (**Single-User** for local analysis or **Multi-User** for collaborative server-based environments).
- **Data Source Selection:** Inputting the media to be analyzed.
- **Ingest Module Configuration:** Running automated analysis plugins.
- **Analysis/Review:** Examining extracted artifacts via the GUI.
- **Reporting:** Generating the final documentation of findings.

## Data Sources

Autopsy supports a wide variety of forensic and virtual image formats:

- **Raw Formats:** Single files (`.img`, `.dd`, `.raw`) or Split files (`.001`, `.002`).
- **Proprietary/Virtual:** EnCase (`.e01`), Virtual Machines (`.vmdk`, `.vhd`).
- **Note:** For split images (e.g., E01/E02), you only need to select the first file; Autopsy automatically chains the subsequent segments.

## Ingest Modules (Analysis Plugins)

Ingest Modules are the "engines" of Autopsy that automate data extraction.

- **Execution Timing:** Can be run during the initial data source import or later by right-clicking a source in the dashboard.
- **Functionality:** They identify specific artifacts (e.g., "Interesting Files Finder," "Keyword Search").
- **Per-Run Settings:** Some modules (marked with a yellow triangle) allow for custom parameters before execution.
- **Ingest Inbox:** A notification area that alerts the analyst to hits/finds in real-time as modules run.

## User Interface Components

The interface is divided into five critical functional areas:

1. **Tree Viewer (Left):** Navigational hierarchy categorized by Data Sources, Views (File Types/MIME), Results (Ingest output), Tags, and Reports.
2. **Result Viewer (Top Right):** Displays contents of the item selected in the Tree Viewer. Features **Table**, **Thumbnail**, and **Summary** tabs.

- _Pro Tip:_ Check **MIME Type** under the Views node to find files hidden with misleading extensions.

3. **Contents Viewer (Bottom Right):** Provides low-level data on a selected file.

- **S (Score):** Red exclamation (Notable) or Yellow triangle (Suspicious).
- **C (Comment):** Indicates analyst or system notes.
- **O (Occurrence):** Shows how often a file has appeared in previous cases (requires Central Repository).

4. **Keyword Search (Top Right):** Allows for **Ad-Hoc** searches or the use of pre-defined **Keyword Lists**.
5. **Status Area (Bottom Right):** Monitors the progress of background tasks and Ingest Modules.

## Specialized Visualization & Reporting

- **Data Sources Summary:** A high-level dashboard providing an overview of findings (OS info, file counts, etc.) before deep-diving.
- **Timeline Tool:** A powerful visualizer for temporal analysis.
- **Counts View:** Bar charts of event density.
- **Details View:** Clustered events (expandable/collapsible).
- **List View:** Chronological table of events.

- **Reporting:** Generates documentation (e.g., HTML, Excel).
- _Practical Use:_ On low-resource systems, generate an HTML report to review findings in a browser to prevent the Autopsy GUI from freezing during heavy data browsing.
