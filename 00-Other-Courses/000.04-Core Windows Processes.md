# Core Windows Processes

This summary outlines the core Windows processes and the diagnostic tools used to differentiate normal system behavior from potential malicious activity.

## Core Diagnostic Tools

Understanding these tools is essential for identifying "outliers" (processes that deviate from expected norms).

- **Task Manager:** Built-in GUI for resource monitoring and process termination. Key columns for analysis include **PID, Command Line, and Image Path**.
- **Process Hacker / Process Explorer:** Advanced alternatives that provide a **Parent-Child process tree view**, which is critical because Task Manager cannot show which process spawned another.
- **Command Line:** Useful native commands include `tasklist`, `Get-Process` (PowerShell), and `wmic`.

## Windows Core Processes Reference

### 1. System (PID 4)

- **Function:** Host for kernel-mode system threads (drivers, kernel code).
- **Normal:** PID is always 4; Single instance; Parent is "System Idle Process" (0).
- **Abnormal:** Multiple instances; Different PID; Parent other than PID 0; Not in Session 0.

### 2. smss.exe (Session Manager)

- **Function:** First user-mode process. Creates new sessions (Session 0 for OS, Session 1 for user). Starts `csrss.exe` and `wininit.exe`.
- **Normal:** Path is `\System32\smss.exe`; Parent is **System (4)**. Master instance stays active; children self-terminate.
- **Abnormal:** More than one active process; Image path outside `System32`; Running as a user instead of SYSTEM.

### 3. csrss.exe (Client Server Runtime)

- **Function:** User-mode side of Windows subsystem. Handles Win32 console, thread creation, and shutdowns.
- **Normal:** Runs in Session 0 and 1; Parent is **smss.exe** (though smss.exe exits, leaving no visible parent in many tools).
- **Abnormal:** An actual persistent parent process; Misspelled names; Running as a user.

### 4. wininit.exe (Windows Initialization)

- **Function:** Launches `services.exe`, `lsass.exe`, and `lsaiso.exe` in Session 0.
- **Normal:** Single instance; Runs as SYSTEM.
- **Abnormal:** Multiple instances; Parent process other than the terminated smss.exe.

### 5. services.exe (Service Control Manager)

- **Function:** Manages system services and device drivers. Parent to `svchost.exe`, `spoolsv.exe`, etc.
- **Normal:** Parent is **wininit.exe**; Single instance; Runs as SYSTEM.
- **Abnormal:** Parent other than `wininit.exe`; Misspelled name; Not running as SYSTEM.

### 6. svchost.exe (Service Host)

- **Function:** Hosts DLL-based services. Services are grouped using the `-k` parameter to save resources.
- **Normal:** Parent is **services.exe**; Many instances; Always contains the **`-k`** flag in the command line.
- **Abnormal:** Parent other than `services.exe`; **Missing `-k` parameter**; Misspellings (e.g., `scvhost.exe`).

### 7. lsass.exe (Local Security Authority)

- **Function:** Enforces security policies, verifies logins, and creates access tokens. Frequent target for credential dumping (e.g., Mimikatz).
- **Normal:** Parent is **wininit.exe**; Single instance; Path is `\System32\lsass.exe`.
- **Abnormal:** Multiple instances; Parent other than `wininit.exe`; Not running as SYSTEM.

### 8. winlogon.exe (Windows Logon)

- **Function:** Handles the Secure Attention Sequence (Ctrl+Alt+Del) and loads user profiles via `userinit.exe`.
- **Normal:** Parent is smss.exe (exits); Runs as SYSTEM; Launches `explorer.exe`.
- **Abnormal:** Shell value in registry is not `explorer.exe`; Actual parent process visible.

### 9. explorer.exe (Windows Explorer)

- **Function:** Provides the user GUI (folders, Start Menu, Taskbar).
- **Normal:** Parent is **userinit.exe** (which exits immediately); Runs as the **Logged-in User** (not SYSTEM).
- **Abnormal:** Running as SYSTEM; Outbound TCP/IP connections; Path outside `C:\Windows`.
