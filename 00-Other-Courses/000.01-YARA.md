# YARA

YARA is a multi-platform tool used by malware researchers to identify and classify files based on **textual or binary patterns**. Think of it as "grep" for malware, using a rule-based syntax to find specific indicators of compromise (IOCs).

## 1. Core Concepts & Syntax

YARA rules are written in a proprietary language. Every rule must contain a **Name** and a **Condition**.

### Anatomy of a Rule

- **Meta:** Descriptive information (author, date, description). Does not affect logic.
- **Strings:** Defined variables (text or hex) to search for within the file.
- **Condition:** The logic that determines a match (e.g., `any of them`, `filesize < 10KB`, or specific string counts).

### Example Rule Logic

```yara
rule Example_Rule {
    meta:
        description = "Detects Hello World strings in small files"
    strings:
        $s1 = "Hello World!"
    condition:
        $s1 and filesize < 10KB
}
```

## 2. Methodologies: Manual vs. Automated

Identifying malware patterns manually is time-consuming (e.g., a simple web shell can contain over 3,000 lines of code). To scale detection, researchers use automated generation and scanning tools.

### Key Tools & Ecosystem

| Tool                 | Creator      | Primary Function                                                                                              |
| -------------------- | ------------ | ------------------------------------------------------------------------------------------------------------- |
| **LOKI**             | Florian Roth | Open-source IOC scanner (Files, Hashes, YARA).                                                                |
| **THOR / THOR Lite** | Florian Roth | Advanced APT scanner with CPU throttling; geared for corporate environments.                                  |
| **Fenrir**           | Florian Roth | A Bash-based IOC scanner for systems with minimal requirements.                                               |
| **yarGen**           | Florian Roth | **Rule Generator.** Creates YARA rules by extracting strings from malware and subtracting "goodware" strings. |
| **YAYA**             | EFF          | "Yet Another Yara Automaton"—Manages and runs multiple rule repositories (Linux only).                        |

## 3. Advanced Integration & Research

- **Modules:** YARA can integrate with **Cuckoo Sandbox** (to generate rules based on dynamic behavior) and **Python PE** (to scan Windows Portable Executable structures).
- **Valhalla:** A high-quality curated feed of YARA rules. It allows analysts to search for rules by ATT&CK techniques, SHA256 hashes, or malware families.
- **Workflow:** 1. Discover a suspicious file.

2.  Scan with **LOKI** using existing signatures.
3.  If undetected, use **yarGen** to create a custom rule based on unique strings.
4.  Verify findings via **Valhalla** for threat intelligence.

## 4. Outcomes

By mastering YARA, security analysts can:

- Perform rapid incident response across multiple endpoints.
- Automate the identification of specific malware families.
- Reduce false positives by filtering out "goodware" strings during rule generation.
