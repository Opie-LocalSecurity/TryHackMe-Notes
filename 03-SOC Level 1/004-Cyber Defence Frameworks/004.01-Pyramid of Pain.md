# Pyramid of Pain

## Concept / takeaway

- **Pyramid of Pain** ranks IOC types by how much pain they cause an attacker when defenders detect & block them.
- Lower levels (hashes, IPs) are easy for attackers to change; higher levels (tools, TTPs) force attackers to rework tooling, tradecraft or training.
- As a defender, prioritize detection that moves you **up the pyramid** (host/network artifacts, tools, TTPs) for long-term impact.

## Levels (bottom → top) — what they are, attacker cost, detection & response

### 1) Hash values — *Trivial*

- **What:** File digests (MD5, SHA-1, SHA-2/SHA-256).
- **Attacker cost to change:** *Very low* — one bit change → different hash.
- **Tools / examples:** VirusTotal, OPSWAT MetaDefender.
- **Detection & response:** Use hash lookups for quick identification; *do not* rely solely on hashes. Use fuzzy hashing (SSDeep) + sandboxing for similarity detection.
- **Practical note:** MD5/SHA-1 are weak for integrity/authenticity.

### 2) IP addresses — *Easy*

- **What:** Source/destination IPv4/IPv6.
- **Attacker cost:** Low — can change IPs, use proxies, cloud, VPNs, fast rotation.
- **Adversary technique:** Fast-Flux (rapidly changing IP set behind a domain).
- **Detection & response:** Block/deny lists, firewall rules, correlate with domain/behavior; monitor DNS patterns and fast-flux characteristics.

### 3) Domain names — *Simple → Moderate*

- **What:** Domains and subdomains used for phishing, C2, payload hosting.
- **Attacker cost:** Moderate — registration & DNS changes required but easily automated via APIs.
- **Tricks to watch for:** Punycode homographs (xn--…), URL shorteners (bit.ly, tinyurl), domain typosquatting.
- **Detection & response:** DNS/HTTP log inspection, reverse DNS, WHOIS, inspect shortened URLs (append “+” where supported), monitor registration patterns and look for Punycode or unusual TLDs.

### 4) Host artifacts — *Annoying*

- **What:** OS-level traces — registry entries, dropped files, suspicious process execution, persistence mechanisms.
- **Attacker cost:** Significant — they may need new tooling or change workflow.
- **Tools / sources:** Any.run sandbox reports, Windows Event Logs, EDR alerts.
- **Detection & response:** Endpoint monitoring/EDR, file system & registry monitoring, alert on anomalous parent/child process relationships, hunt for dropped files and known malicious behaviours.

### 5) Network artifacts — *Annoying*

- **What:** Network-level indicators — User-Agent strings, URI patterns, HTTP POST bodies, C2 protocols.
- **Attacker cost:** Moderate → higher (must change C2/communication patterns).
- **Tools:** Wireshark/TShark, IDS (Snort), proxy/webserver logs, PCAP analysis.
- **Detection & response:** Parse PCAPs, extract UA/URIs, create IDS signatures, block suspicious UA strings or patterns, detect abnormal beaconing/POST payloads.

### 6) Tools — *Challenging*

- **What:** Binaries, scripts, dropper tools, custom malware, maldocs and utilities attackers use.
- **Attacker cost:** High — replacing or reengineering tools is time/resource consuming.
- **Tools / feeds:** MalwareBazaar, Malshare, YARA rules, SOC Prime TDM marketplace.
- **Detection & response:** YARA rules, signatures, fuzzy hashing (SSDeep) to catch variants, share detection rules, collect samples for analysis and create persistent detections.

### 7) TTPs (Tactics, Techniques & Procedures) — *Tough / Apex*

- **What:** Entire attack choreography (MITRE ATT&CK techniques), e.g., phishing, lateral movement, Pass-the-Hash.
- **Attacker cost:** Very high — forces changes in tradecraft, playbooks, or target selection.
- **Detection & response:** Focus on behavioral detection (event-log correlation, sequence detection across endpoints & network), map alerts to MITRE ATT&CK, quickly remediate (isolate host, reset creds, stop lateral movement). This yields the largest strategic advantage.

## Key tools & resources mentioned

- **Hash / sample lookup:** VirusTotal, OPSWAT MetaDefender.
- **Sandbox / dynamic analysis:** Any.run (networking/DNS/HTTP tabs, connections).
- **Sample feeds / repos:** MalwareBazaar, Malshare.
- **Detection rules / marketplace:** SOC Prime Threat Detection Marketplace.
- **Fuzzy hashing:** SSDeep (similarity matching).
- **Network analysis / IDS:** Wireshark/TShark, Snort.
- **Rule authoring / signatures:** YARA, IDS/EDR rules.
- **Framework for TTPs:** MITRE ATT&CK.

## Practical, actionable guidance (quick wins)

- Use **hash lookups** for fast triage but pair with fuzzy hashing and sandbox results.
- Correlate **IP + domain + network behaviour + host artifacts** before acting (reduces false positives).
- Prioritize telemetry that exposes **TTPs** (sequences of actions) — build detections mapped to MITRE ATT&CK.
- Invest in EDR + network visibility (PCAP/IDS logs) so you can detect host + network artifacts and link to higher-level TTPs.
- Maintain YARA rules and fuzzy-hash similarity detection to catch variants of known tools.
