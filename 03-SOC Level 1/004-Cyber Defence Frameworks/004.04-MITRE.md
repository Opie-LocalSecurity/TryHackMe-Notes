# MITRE

## MITRE (context)

- Not-for-profit R&D org supporting safer-world mission; active in cyber, AI, health, space.
- Provides community knowledge bases, tools, and frameworks that standardize how we describe, detect, and counter adversary behavior.

## ATT&CK® Framework (what & why)

- **What:** Public knowledge base of real-world adversary **Tactics**, **Techniques**, and **Procedures (TTPs)** with unique IDs.
    - **Tactic = why** (goal), **Technique = how** (method), **Procedure = implementation** (exact steps/tools).
- **Scope/Evolution:** Started with Windows; now covers **Enterprise (Windows/macOS/Linux/cloud)**, plus **Mobile** and **ICS**.
- **Why it matters:** Creates a **shared language** to compare incidents, map intel to detections, and align red/blue activities.

## ATT&CK Matrix & Navigator (how to use)

- Matrix shows **tactics (columns)** with **techniques & sub-techniques (cells)**.
- **Navigator** lets you **annotate/overlay** coverage, groups, detections.
- Example: **Reconnaissance → Active Scanning (T1595)** → sub-techniques: **IP block scanning, Vulnerability scanning, Wordlist scanning**.
- Technique pages include: description, IDs, **procedure examples (groups/software/campaigns)**, **mitigations**, **detections**, references.

## Operational Use (who uses ATT&CK & outcomes)

- **CTI teams:** Map reports to TTPs to build actionable group profiles.
- **SOC analysts:** Add ATT&CK context to alerts; prioritize triage.
- **Detection engineers:** Align **SIEM/EDR** rules to techniques for measurable coverage.
- **Incident responders:** Reconstruct timelines by tactic/technique for clearer attack paths.
- **Red/Purple teams:** Build realistic **emulation plans** aligned to known group behaviors.

## Mapping in Action (example)

- Post-incident, map each attack stage to ATT&CK to expose gaps.
- **Mustang Panda (G0129):** Prefers **phishing** for Initial Access, **scheduled tasks** for Persistence, **obfuscation** for Defense Evasion, **ingress tool transfer** for C2. Use its **group page + Navigator layer** to visualize technique set and prioritize defenses.

## ATT&CK for Threat Intelligence (scenario workflow)

1. From **ATT&CK Groups**, pick actors targeting your sector (e.g., **aviation**, **cloud** migration context).
2. Load the group’s **Navigator layer**; note tactics/techniques used.
3. Compare against your **current detections & controls**; mark **coverage vs. gaps**.
4. Drive **detection engineering** and **playbook updates** from the mapped gaps.

## Cyber Analytics Repository (CAR) (detections you can run)

- **What:** Curated detection analytics mapped to ATT&CK; includes **data model**, **pseudocode**, and tool-specific **queries** (e.g., **Splunk**, **LogPoint**).
- **Use:** Translate TTPs into **queryable detections**; some analytics include **unit tests** for validation.
- Example: **CAR-2020-09-001 (Scheduled Task – File Access):**
    - Detects creation/changes of scheduled-task files (persistence).
    - Provides pseudocode and example SIEM queries.
- **Tip:** Use the **CAR Navigator layer** to see which ATT&CK techniques have out-of-the-box analytics.

## MITRE D3FEND (how to stop attacks)

- **What:** Defensive counterpart to ATT&CK; standardized **defensive techniques** organized by seven tactics (**Model, Harden, Detect, Isolate, Deceive, Evict, Restore**).
- **Value:** Maps **defenses ↔ ATT&CK techniques** + **digital artifacts** to design mitigations intentionally.
- Example: **Credential Rotation (D3-CRO):** Regularly rotate credentials to blunt reuse of stolen creds; includes implementation considerations and related attacker techniques.

## Other MITRE/CTID Resources (to practice & automate)

- **Adversary Emulation Library:** Step-by-step **emulation plans** for real groups; great for purple-team exercises and validation.
- **Caldera:** Automated **adversary emulation** platform using ATT&CK; supports offensive/defensive ops for continuous testing.
- **AADAPT:** Matrix for threats to **digital asset/payment** tech (blockchain, wallets, smart contracts).
- **ATLAS:** Matrix for threats to **AI/ML systems** (real-world techniques, vulns, mitigations).

## Quick Application Checklist

- **Identify threats:** Pick sector-relevant groups from **ATT&CK Groups**.
- **Visualize behavior:** Load **Navigator** layers; tag techniques seen vs. suspected.
- **Detect:** Pull matching **CAR** analytics; adapt queries to your SIEM/EDR; add unit tests when available.
- **Mitigate:** For each risky technique, choose **D3FEND** controls (e.g., hardening, deception, eviction).
- **Validate:** Run **Caldera** or emulation plans; measure detections/gaps and iterate.

This gives you a compact map from **intel → detection → defense → validation** using MITRE’s ecosystem.
