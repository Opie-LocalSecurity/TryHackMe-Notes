# Unified Kill Chain

## **What a “Kill Chain” is**

- Military-origin concept adapted to cybersecurity: a staged model describing how an attacker progresses from initial reconnaissance to completing their objective.
- Purpose: map attacker behaviour so defenders can place controls to prevent, detect, or disrupt attacks at each stage.

## **Threat modelling (short)**

- Goal: reduce risk by systematically identifying assets, threats, vulnerabilities, and mitigations.
- Core steps:
    1. Inventory & classify systems/apps (criticality, sensitive data).
    2. Identify vulnerabilities and likely exploit paths.
    3. Design and prioritise mitigations (technical + process).
    4. Implement policies to prevent recurrence (SDLC, user training, etc.).
- Common frameworks used in threat modelling: **STRIDE**, **DREAD**, **CVSS** (and others).

## **Unified Kill Chain (UKC) — overview**

- Author: Paul Pols. Modern, detailed kill-chain model (officially **18 phases**).
- Designed to complement (not replace) frameworks like Lockheed Martin’s and MITRE ATT&CK.
- Key advantages:
    - More granular (18 phases) — models the full attack lifecycle.
    - Accounts for iterative behaviour (phases repeat; attackers loop back).
    - Covers attacker motivation and post-exploitation activity, not just initial access.

## UKC phases (grouped for clarity)

### Phase: **In** — *Initial foothold* (attacker gains access)

- **Reconnaissance** (MITRE TA0043): passive/active info gathering — services, topology, employee lists, credential leaks.
- **Weaponization** (TA0001): prepare infrastructure (C2 servers, payloads).
- **Social engineering** (TA0001): phishing, impersonation, malicious attachments.
- **Exploitation** (TA0002): abusing vulnerabilities to execute code (e.g., web upload → reverse shell).
- **Persistence** (TA0003): backdoors, services, scheduled tasks to retain access.
- **Defence evasion** (TA0005): bypass AV, WAFs, IDS/IPS, logging/visibility.
- **Command & Control (C2)** (TA0011): establish comms channel for remote control and data movement.
- **Pivoting** (TA0008): use compromised host to reach internal/non-public systems.

### Phase: **Through** — *Network propagation & consolidation*

- **Pivoting** (reiterated): staging point / distribution for malware.
- **Discovery** (TA0007): build internal knowledge — users, privileges, shares, configs.
- **Privilege escalation** (TA0004): upgrade access (local admin → SYSTEM/ROOT).
- **Execution** (TA0002): deploy malware, scheduled tasks, remote trojans.
- **Credential access** (TA0006): keylogging, credential dumping, harvest legit creds.
- **Lateral movement** (TA0008): move across environment using stolen creds and exploits.

### Phase: **Out** — *Action on objectives (goal achievement)*

- **Collection** (TA0009): gather target data (files, browser data, audio/video, mail).
- **Exfiltration** (TA0010): compress/encrypt and steal data via C2 or other channels.
- **Impact** (TA0040): actions to affect availability/integrity — ransomware, wipes, defacement, DoS.
- **Objectives**: final attacker motivations — financial gain, espionage, reputation damage, disruption.

## Practical takeaways for defenders

- Map controls to kill-chain stages (prevent at recon/initial access, detect in C2/persistence, respond during lateral movement).
- Use threat modelling to prioritize defenses based on asset criticality and likely attack paths.
- Consider UKC for a more granular playbook and to model iterative attack behaviour.
- Integrate people/process (training, SDLC) with technical controls (patching, MFA, EDR, segmentation, logging/monitoring).
