# Alert Triange with Elastic

## Goal

Triage multiple alerts, correlate IIS web logs with Windows Security and Sysmon to confirm a compromise, reconstruct the attack path, and gather evidence for escalation.

## Environment Setup (Kibana)

- **Data view:** `Alert Triage With Elastic`
- **Time range:** `Entire data range`
- **Starter IIS filter (Q2):** `_index:weblogs`
- **Technique:** Add fields as table columns for quick triage (use the **+** beside fields).

## Web Attack Triage (IIS)

### ProxyLogon-style POSTs (High severity)

- **Query:** `_index:weblogs and client.ip:203.0.113.55 and http.request.method:POST`
- **Columns:** `client.ip`, `user.agent`, `http.request.method`, `url.path`, `http.response.status_code`
- **Why:** Confirm automated POSTs to `proxyLogon.ecp` from `203.0.113.55`; user-agent + path suggest **ProxyLogon exploitation** → **True Positive** candidate.

### Web shell GETs (Follow-on, 7 min later)

- **Query:** `_index:weblogs and client.ip:203.0.113.55 and http.request.method:GET and errorEE.aspx`
- **Sort:** `Old → New`
- **Evidence:** `url.path` shows `errorEE.aspx?cmd=...` patterns → **web shell command execution confirmed** → escalate both web alerts as **True Positive**.

## Account Activity (Windows Security + Sysmon)

### Suspicious Administrator logon (Event ID 4624)

- **Query:** `@timestamp >= "2025-07-20T05:11:22" and winlog.event_id:4624 and host.name:winserv2019.some.corp and winlog.event_data.TargetUserName:Administrator`
- **Columns:** `winlog.event_id`, `host.name`, `winlog.event_data.TargetUserName`, `winlog.logon.type`, `winlog.event_data.IpAddress`
- **Finding:** Admin logon at alert time **from 203.0.113.55** (same attacker IP) → ties web compromise to host access.

### Process creation after logon (Sysmon Event ID 1)

- **Query:** `@timestamp >= "2025-07-20T05:11:22" and winlog.event_id:1 and user.name:Administrator`
- **Columns:** `user.name`, `process.parent.name`, `process.command_line`
- **Finding:** Normal session-init chain present; confirms session context started by **Administrator**.

### New user creation (Security: User Account Management)

- **Query:** `@timestamp >= "2025-07-20T05:13:10.000" and winlog.channel:Security and winlog.task:User Account Management`
- **Columns:** `winlog.event_id`, `winlog.task`, `message` (sort `Old → New`)
- **Finding:** **New local user created** soon after Admin logon (backdoor account setup). Messages reveal which account and groups.

## Command-Line & Privilege Changes

### Cmd.exe child activity (Sysmon)

- **Scope query:** `@timestamp >= "2025-07-20T05:13:15" and process.parent.name:cmd.exe and user.name:Administrator`
- **Columns:** `process.command_line`, `process.name`, `process.parent.name`
- **Finding:** Shows exact commands (e.g., `net` user/group ops) executed under **cmd.exe** by **Administrator**.

### Correlate group changes (Security 4732 + Sysmon)

- **Joint query:** `@timestamp >= "2025-07-20T05:13:15" and (winlog.event_id:4732 or process.parent.name:cmd.exe)`
- **Purpose:** Align **Security 4732** (user added to local groups) with Sysmon process lineage from `cmd.exe`.
- **Finding:** New account added to **multiple groups** (e.g., Administrators/Remote Desktop Users) → **privilege expansion** confirmed.

## PowerShell Recon (Module: PowerShell, Code 4104)

- **Query:** `@timestamp >= "2025-07-20T05:13:15" and event.module:powershell and event.code:4104`
- **Column:** `powershell.file.script_block_text` (sort `Old → New`)
- **Finding:** `whoami` and `whoami /priv` → **post-exploitation discovery** by attacker (validates continued hands-on activity).

## Unalerted Data Exfil Prep (RAR usage)

- **Query:** `process.name: "Rar.exe"`
- **Why:** Detect **legitimate tool misuse** by the newly created user, check timing/parent process, and correlate to account and session.
- **Decision:** If parent is `cmd.exe`/PowerShell from attacker session and timing follows privilege changes → treat as **malicious staging/exfil** despite no alert.

## Reconstructed Timeline (Jul 20, 2025)

1. **~04:38** – Automated `POST` to `proxyLogon.ecp` from **203.0.113.55** (IIS).
2. **~04:45** – `GET` to `errorEE.aspx?cmd=...` → **web shell execution** (IIS).
3. **05:11** – **4624** logon: **Administrator** from **203.0.113.55** (host: `winserv2019.some.corp`).
4. **05:13** – **New local user created**; **4732** adds user to groups (via `cmd.exe`).
5. **≥05:13** – **PowerShell** discovery (`whoami`, `whoami /priv`).
6. **Later** – **Rar.exe** executed by the backdoor context (potential staging/exfil), no alert.

## Key Indicators & Evidence

- **Attacker IP:** `203.0.113.55` (web logs and 4624 source IP).
- **Malicious paths:** `/proxyLogon.ecp`, `/errorEE.aspx?cmd=...` (web shell).
- **Accounts:** `Administrator` (initial access), **newly created local user** (persistence/privilege).
- **Events:** Security **4624**, **4732**; Sysmon **1**; PowerShell **4104**.
- **Processes:** `cmd.exe` (parent), `net` commands, `Rar.exe` (LOLBIN/tool abuse).

## Methodology Pattern (Repeatable Playbook)

- **Start with alert → scope via targeted queries** (IIS or Windows/Sysmon).
- **Add essential columns** to make context obvious (IP, user, command line, path, status).
- **Pivot by time and principal** (use alert timestamp as a lower bound).
- **Correlate across sources** (IIS ↔ 4624 ↔ Sysmon 1 ↔ Security 4732 ↔ PowerShell 4104).
- **Validate intent** (web shell cmd params, privilege/group changes, discovery, staging).

## Outcome & Escalation Rationale

- **Verdict:** **Confirmed compromise (True Positive)** with clear kill chain:
    - External exploitation (**ProxyLogon** pattern) → **Web shell** → **RDP/Admin logon** → **Persistence & privilege changes** → **Recon** → **Data staging tool (RAR)**.
- **Evidence pack (what to export/share):**
    - Screenshots or CSV excerpts of the above queries/tables.
    - Full `url.path` entries showing `cmd=` parameters.
    - Security **4624/4732** event details (account, source IP, groups).
    - Sysmon **1** `process.command_line` for `cmd.exe` children and `Rar.exe`.
    - PowerShell 4104 script blocks (`whoami`, `/priv`).

**Next actions (recommend):**

- Contain source IP, disable newly created user, reset Administrator credentials, collect memory + disk forensics, review outbound traffic for potential exfil tied to `Rar.exe`, patch Exchange/IIS (ProxyLogon), and hunt for additional web shells.
