# Web Security Essentials

## 🌐 **Why Web?**

- **Evolution:**
    - 1990s – Desktop apps dominate (limited connectivity).
    - 2000s – Dynamic web apps emerge (email, social media, banking).
    - 2010s – Cloud & SaaS expansion; now nearly everything runs in browsers.
- **Benefits:** Accessibility, faster updates, better compatibility, reduced local resources.
- **Security Tradeoffs:** Greater attack surface due to constant exposure and global access.

## 🔒 **Security Perspective**

- **Why risk increases:** Web apps are public-facing and often connect to critical backend systems (databases, cloud).
- **Risks:**
    - **For Owners:** Constant exposure, evolving threats, responsibility for user data.
    - **For Users:** Risk of data theft, privacy loss, and compromised accounts.
- **Examples:**
    - *Equifax (2017)* – Apache vulnerability exploited → massive data breach.
    - *Capital One (2019)* – Misconfigured WAF exposed customer data.

## 🧩 **Web Infrastructure**

- **Request–Response Cycle:** Browser → Web Server → Response → User.
    
    Attackers exploit this flow to perform DoS, bypass controls, or inject commands.
    
- **Core Components:**
    1. **Application** – Code, interface, and logic.
    2. **Web Server** – Handles incoming requests (e.g., Apache, Nginx, IIS).
    3. **Host Machine** – Underlying OS hosting the web service.

## 🛡️ **Protecting the Web**

### **1. Application-Level**

- Secure coding (no hardcoded secrets, error handling).
- Input validation/sanitization.
- Access control (role-based).

### **2. Web Server-Level**

- Logging (track and analyze requests).
- Web Application Firewall (WAF) for traffic filtering.
- CDN integration for traffic distribution and masking.

### **3. Host Machine-Level**

- Least privilege for services.
- System hardening (disable unused ports/services).
- Antivirus for endpoint protection.

### **Universal Best Practices**

- Strong authentication.
- Continuous patch management across all layers.

## 📊 **Logging**

- **Access Logs** record: IP, timestamp, URL, response code, user agent.
- Used for **incident response and attack reconstruction**.
- Common HTTP methods:
    - `GET` → retrieve data.
    - `POST` → submit data (e.g., logins).

## 🌍 **Defense Systems**

### **1. Content Delivery Network (CDN)**

- **Function:** Distribute cached content via global edge servers for speed and resilience.
- **Security Benefits:**
    - IP masking (hides origin).
    - DDoS absorption.
    - Enforced HTTPS (TLS).
    - Integrated WAF (e.g., Cloudflare, AWS CloudFront, Azure Front Door).

### **2. Web Application Firewall (WAF)**

- **Purpose:** Inspects and filters HTTP traffic before reaching the web app.
- **Types:**
    - Cloud-based (reverse proxy).
    - Host-based (local software).
    - Network-based (appliance).
- **Detection Methods:**
    
    
    | Method | Description | Example |
    | --- | --- | --- |
    | Signature-based | Matches known patterns | `sqlmap/1.8.1` in User-Agent |
    | Heuristic | Context-aware analysis | Long encoded strings |
    | Behavioral | Detects anomalies | Rapid login attempts |
    | IP Reputation | Blocks based on threat intel | Requests from flagged regions |

### **3. Antivirus (AV)**

- **Purpose:** Detects known malicious files on endpoints/servers.
- **Detection:** Signature-based (file comparisons).
- **Role in Web Security:** Detects web shells, malware uploads, and post-exploitation tools.
- **Limitation:** Does not secure the application layer — part of a **defense-in-depth** strategy.

## ✅ **Key Takeaways**

- The web’s ubiquity brings convenience and persistent exposure.
- Security must span **application, server, and host layers**.
- **Defense-in-depth**: Combine WAFs, CDNs, AVs, and proper coding/patching to minimize risk.
- **Monitoring and logging** are as vital as preventive controls for detection and response.
