# Splunk: Basics

## **Splunk Overview**

Splunk is a **Security Information and Event Management (SIEM)** platform used to **collect, index, and analyze machine data** in real time. It enhances visibility into network activity and accelerates threat detection.

## **Core Components**

### **Splunk Forwarder**

- A lightweight agent installed on monitored endpoints.
- Collects data and sends it to the Indexer.
- Designed for minimal resource use.
- **Typical data sources:**
    - Web servers (web traffic)
    - Windows (Event Logs, PowerShell, Sysmon)
    - Linux hosts (system logs)
    - Databases (connection requests, responses, errors)

### **Splunk Indexer**

- **Processes and stores data** received from forwarders.
- Parses data into **field-value pairs** and stores it as searchable events.
- Enables efficient querying and analysis.

### **Splunk Search Head**

- The user interface for performing searches using **SPL (Search Processing Language)**.
- Sends search requests to the Indexer and retrieves relevant events.
- Allows users to **visualize results** using tables, pie charts, bar charts, etc.
- Part of the default **Search & Reporting App**.

## **Navigating Splunk**

### **Home Screen Sections**

1. **Splunk Bar**
    - Top navigation menu with options:
        - *Messages* – System notifications
        - *Settings* – Configure instance parameters
        - *Activity* – Track running search jobs
        - *Help* – Access documentation/tutorials
        - *Find* – Search across apps
    - Also allows switching between installed Splunk apps.
2. **Apps Panel**
    - Displays installed apps (default: *Search & Reporting*).
    - Apps can also be switched via the Splunk Bar.
3. **Explore Splunk**
    - Quick access to:
        - Add new data
        - Install new apps
        - View documentation
4. **Home Dashboard**
    - Central space for viewing dashboards.
    - Users can create, customize, and view their own dashboards under the **“Yours”** tab.

## **Adding Data to Splunk**

- Splunk can ingest **any type of machine data** (e.g., event logs, web logs, firewall logs).
- Data is **transformed into individual events** for indexing and search.
- **Common data categories:** Security, Web, System, Application, Network, etc.

### **Data Upload Process (Example: VPN Logs)**

To add data manually:

1. **Select Source** – Choose the log file.
2. **Select Source Type** – Define data format (e.g., JSON, syslog).
3. **Input Settings** – Choose an index and hostname.
4. **Review** – Verify configuration.
5. **Done** – Upload completes; data becomes searchable.

## ✅ **Outcome:**

After data ingestion, users can search, correlate, and visualize event data to monitor network activity, detect anomalies, and generate security insights in real time.
