# Elastic Stack: The Basics

## 🧩 **Elastic Stack (ELK) Overview**

**Purpose:**

Originally built for data storage, search, and visualization; now widely used in **Security Operations Centers (SOC)** as a lightweight **SIEM** solution.

**Core Components:**

1. **Elasticsearch** –
    - Full-text search and analytics engine for JSON documents.
    - Stores, indexes, and correlates data; accessible via REST API.
2. **Logstash** –
    - Data processing pipeline that ingests, filters, and outputs data.
    - Config split into:
        - **Input:** data sources
        - **Filter:** normalization/field extraction
        - **Output:** destinations (e.g., Elasticsearch, Kibana)
3. **Beats** –
    - Lightweight data shippers (agents) on hosts.
    - Each beat collects specific data (e.g., *Filebeat* for logs, *Winlogbeat* for Windows events, *Packetbeat* for network data).
4. **Kibana** –
    - Web-based visualization and analytics tool for Elasticsearch.
    - Used to explore, search, and build dashboards.

**Data Flow:**

- **Beats → Logstash → Elasticsearch → Kibana**
    
    Beats collect → Logstash processes → Elasticsearch stores → Kibana visualizes.
    

## 🔍 **Kibana: Discover Tab**

**Purpose:**

Main workspace for SOC analysts to view, search, and filter logs.

**Key Elements:**

- **Logs View:** Each row = one log entry.
- **Fields Pane:** Lists parsed fields; allows inclusion/exclusion in view.
- **Index Pattern:** Defines which Elasticsearch indices Kibana queries (e.g., `vpn_connections`).
- **Search Bar:** Enter queries or filters (uses KQL).
- **Time Filter & Timeline:** Restrict data by date/time; detect log spikes.
- **Add Filter:** Apply specific field-based filters without typing queries.
- **Table Creation:** Select important fields to simplify raw log view; can be saved for reuse.

## 🔠 **KQL (Kibana Query Language)**

**Used In:** Kibana Search Bar (Discover tab)

### 1. **Free Text Search**

- Searches across all fields.
    - Example: `security` → finds logs with the term “security”.
- Supports wildcards () for partial matches:
    - `United*` → matches “United States”, “United Kingdom”.

### 2. **Logical Operators**

- **AND:** `"United States" AND "Virginia"`
- **OR:** `"United States" OR "England"`
- **NOT:** `"United States" AND NOT "Florida"`

### 3. **Field-Based Search**

- Syntax: `Field: Value`
    - Example: `Source_ip : 238.163.231.224 AND UserName : Suleman`
- Auto-suggests available fields.

## 📊 **Creating Visualizations**

**Purpose:**

Transform log data into graphical representations (tables, bar charts, pie charts).

**Process:**

1. From **Discover tab**, choose a field → select **Visualize**.
2. Pick chart type (table, pie, bar, etc.).
3. Use **Correlation** to relate multiple fields (e.g., Source_IP vs Source_Country).
4. **Save Visualization:** Add title, description, and store in library.

**Example Use Case:**

Visualize failed VPN connection attempts by user and IP.

## 🖥️ **Creating Dashboards**

**Purpose:**

Combine multiple visualizations and saved searches for real-time visibility.

**Steps:**

1. Go to **Dashboard tab → Create Dashboard**.
2. **Add from Library** existing visualizations and searches.
3. Arrange components and **Save** dashboard.

**Example:**

A dashboard showing VPN connection patterns, failed logins, and source geolocation.

## ✅ **Summary of SOC Analyst Responsibilities**

- Use **Kibana (Discover tab)** to analyze logs and detect anomalies.
- Perform **KQL searches** for targeted investigations.
- Build **visualizations** for insights.
- Create **dashboards** for ongoing monitoring and reporting.
