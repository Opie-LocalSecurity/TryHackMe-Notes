# Introduction to EDR

## What it is

- **Endpoint Detection & Response (EDR):** Host-level security that continuously **monitors**, **detects**, and **responds** to threats on endpoints—on or off the corporate network.
- Goal: Deep visibility + rapid containment of advanced threats (incl. fileless and living-off-the-land attacks).

## Why EDR vs. Antivirus (AV)

- **AV:** Primarily signature-based; stops known bad files at ingress.
- **EDR:** Behavior-focused + context-rich telemetry and response.
- Scenario contrast (phishing → macro → PowerShell → injection → C2):
    - AV likely misses unsigned/obfuscated steps and in-memory activity.
    - EDR observes abnormal **parent–child** process chains, obfuscated script runs, **process injection**, and suspicious outbound connections, then alerts with full attack chain and enables response.

## Core Pillars / Features

- **Visibility**
    - Collects and visualizes endpoint activity (process, registry, files, user actions, network).
    - Process trees, timelines, and historical data aid investigations and hunting.
- **Detection**
    - Signature + behavior + anomaly + **IOC** matching.
    - **MITRE ATT&CK mapping** for each detection.
    - ML models surface complex/multi-stage/fileless patterns.
- **Response**
    - Centralized actions: **isolate host**, **terminate process**, **quarantine files**, **remote shell** (RTR), scripted actions, artifact collection.

## How EDR Works

- **Agents (Sensors):** Deployed on endpoints; capture telemetry and perform local detections.
- **EDR Console:** Correlates telemetry with threat intel and analytics/ML; assigns severities; presents context (files, processes, registry, network).
- **Analyst Workflow:** Triage by severity → validate (true/false positive) → respond from console → document/close.

## Telemetry Collected (examples)

- **Processes:** starts/stops, parent–child relationships, image hashes.
- **Network:** connections, ports, destinations (potential C2/lateral movement).
- **Command line:** CMD/PowerShell usage, obfuscation indicators.
- **Files/Folders:** creation/modification/deletion, staging/ransomware behaviors.
- **Registry:** autoruns, configuration changes tied to persistence.
- Rich historical storage enables root-cause analysis and timeline reconstruction.

## Detection Techniques (at a glance)

- **Behavioral:** Unusual chains (e.g., `winword.exe` → `powershell.exe`).
- **Anomaly:** Deviations from learned baselines.
- **IOC Matching:** Hashes, domains, IPs from threat intel.
- **MITRE Mapping:** Tactics/Techniques labeled (e.g., Persistence → Scheduled Task/Job).
- **Machine Learning:** Pattern recognition across benign-looking steps.

## Response Capabilities

- **Containment:** Network **isolation** of infected hosts.
- **Disruption:** **Terminate** malicious processes; **quarantine** files.
- **Hands-on:** **Remote shell** for custom scripts, deeper inspection.
- **Forensics:** Pull **memory dumps**, **event logs**, **registry hives**, targeted folders.

## Outcomes & Limitations

- **Outcomes:** Faster detection, rich context, organization-wide visibility, swift containment, improved investigations and threat hunting.
- **Limitations:** Host-focused—**not** a replacement for network-level monitoring; best when integrated with broader controls (Firewalls, Email Security, IAM, DLP) and centralized in a **SIEM**.

## Representative EDR Tools

- CrowdStrike Falcon, SentinelOne ActiveEDR, Microsoft Defender for Endpoint, OpenEDR, Symantec EDR.
    
    *Architectures are similar; features and analytics depth vary.*
    

**Remember:** EDR elevates from “is this file known bad?” to “is this behavior normal for this endpoint?”—and gives you the buttons to stop it.
