# Phishing Emails in Action

## Common attacker techniques (seen across samples)

- **Sender spoofing / mismatched addresses** — display name (e.g., [service@paypal.com](mailto:service@paypal.com)) differs from actual envelope sender (gibberish@domain).
- **HTML impersonation** — crafted HTML that visually matches PayPal, DHL, OneDrive/Adobe, Netflix, Apple.
- **Urgency / fear / expiration** — “your account suspended”, “download expires today”, “cancel order” to force quick action.
- **Link manipulation** — shortened URLs, redirect chains, links whose visible text doesn’t match the real destination.
- **Credential harvesting** — fake login pages (SharePoint/OneDrive/Adobe lookalikes) that capture credentials.
- **Tracking pixels** — tiny images (Tracking.png) that notify the sender when an email is opened.
- **Attachments with payloads** — Office/Excel templates or PDFs with embedded links or macros.
- **Typos / poor grammar / typosquatting** — misspellings in brand names, domains or emails are warning signs.
- **BCC/blank bodies** — BCCed recipients or empty body with only an attachment (attempt to evade scrutiny).

## Per-sample highlights (one line each)

- **Cancel your PayPal order** — spoofed sender, shortened URL that redirects (investigate with an URL expander), single “Cancel order” CTA.
- **Track your package** — tracking image used as pixel tracker; links point to suspicious domain; images were blocked by provider (good).
- **View document / Fax (OneDrive → Adobe → login)** — multi-stage redirect to fake Microsoft/Adobe pages; credential harvesting and grammatical errors; fake success/error responses to hide exfiltration.
- **Update your payment details (Netflix)** — spoofed sender, urgent suspension language, attachment (PDF) containing embedded “Update Payment” link (attachment used to bypass link scanning).
- **Your recent purchase (Apple)** — spoofed sender, recipient BCCed, blank body with .dot attachment that contains an image linking to app/ios keywords.
- **DHL shipping notice** — impersonated DHL look, attachment (Excel) that executes payload → error shown (malicious behavior).

## Investigation methods shown

- **Inspect raw email source / headers** — compare `From:` display name vs. envelope sender and `Received:` routing to find origin.
- **Hover / reveal link destinations** — view actual URL target; if blocked by client, inspect raw HTML.
- **Expand shortened URLs** — use URL unshorteners to see final redirect.
- **Check for embedded images / pixels** — look for tiny external image requests (tracking.png).
- **Open attachments in a safe sandbox** — analyze attachments with tools like Any.Run (sample referenced) rather than a production machine.
- **Follow redirect chain in a controlled environment** — to observe credential collection flows and landing pages.

## Top red flags to teach/train on

- Sender address that doesn’t match the brand or your account.
- Any urgent demand to “act now” (download, cancel, update billing).
- Links that shorten/redirect, or visible text differs from actual href.
- Requests to re-enter credentials via an email link (especially for non-auth events).
- Unexpected attachments, especially with macros, .dot, or .xlsm.
- Poor grammar, strange phone numbers, or odd domain names.
- Email images blocked by the client — treat as potential trackers until verified.

## Immediate defensive actions (operational checklist)

- Do **not** click links or open attachments in suspicious messages.
- Verify by logging in directly via the official site (type URL or use a bookmark), never via the email link.
- Inspect headers / `Return-Path` and `Received` lines for mismatches.
- Expand shortened URLs before visiting; scan final URL with a URL reputation tool.
- Keep images blocked by default and don’t enable auto-download of remote content.
- Use MFA/2FA everywhere — prevents credential replay even if passwords are exfiltrated.
- Run attachments in a sandbox (Any.Run, VirusTotal, isolated VM) if you must analyze.
- Report the phishing email to your org’s security team and the impersonated brand (e.g., PayPal, Microsoft, Apple).

## One-line takeaway

Phishers combine **visual impersonation + urgency + link/attachment trickery**; always verify the sender and destination independently, treat unexpected attachments as hostile, and use headers/sandboxing/URL expansion as your investigation tools.
