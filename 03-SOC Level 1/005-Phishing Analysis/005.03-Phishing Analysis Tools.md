# Phishing Analysis Tools

## Immediate safety rule

- **Never click links or open attachments** in the original email. Work from copies (save attachments, extract headers/body as raw text).

## Email *header* artifacts (collect these first)

- **Sender email address**
- **Sender IP address(es)** (from Received: hops)
- **Reverse DNS (PTR) lookup** for the sender IP
- **Reply-To address** (if present) and **Return-Path**
- **Recipient(s)** (To / CC / BCC as available)
- **Subject line**
- **Date / time** (use the timestamp(s) in headers)
- **Full raw header** (copy/paste the entire header into analysis tools)

Why: some items (IP, Return-Path, Received hops) are only in the raw header and reveal mail routing and possible relay abuse.

## Email *body* and attachment artifacts

- **All URLs** found in the body (if shorteners used, resolve to real URL)
- **Root domain(s)** of URLs (analyze domain reputation separately)
- **Attachment file name(s)**
- **Attachment hash(es)** — prefer **SHA256** (MD5 if needed). Example command:
    
    ```bash
    sha256sum "Double Jackpot Slots Las Vegas.dot"
    # -> c650f397...  Double Jackpot Slots Las Vegas.dot
    ```
    
- **Extracted strings / visible HTML source** (use HTML view or saved source)
- **Screenshots / text copy** of the rendered email (for evidence)
- **Any unusual headers or X- headers** called out in the header analysis

Why: URLs give landing pages & phishing domains; hashes let you check file reputation services without uploading the file.

## Safe extraction / parsing techniques

- Save attachments (e.g., Thunderbird “Save”) rather than opening them.
- Extract raw email source (client option “Show original” / “View source”) and work on that copy.
- Use offline or sandboxed environments (or dedicated online sandboxes) to interact with URLs/files — don’t test on production hosts.

## Recommended tools and what they give you

- **Header analysis**
    - Google Admin Toolbox — *Messageheader* (paste raw header → shows routing, delays, hop data).
    - Message Header Analyzer (mha.azurewebsites.net)
    - mailheader.org
- **IP / domain / URL reputation**
    - **ipinfo.io** — geolocation / ASN info for IPs.
    - **urlscan.io** — remote URL scan and automated screenshot; shows contacted hosts and resources.
    - **Talos Reputation Center** — domain/IP/file reputation lookups.
- **URL extraction / parsing**
    - URL Extractor (convertcsv.com) or **CyberChef** (Extract URLs recipe).
- **File/hash reputation**
    - **VirusTotal** — file/hash and URL scans, community detections.
    - **Talos File Reputation** — Cisco file reputation lookup by hash.
    - **Reversing Labs** — file reputation service.
- **Malware sandboxing** (dynamic analysis)
    - **Any.run**, **Hybrid Analysis**, **Joe Sandbox** — execution traces, network calls, IOCs.
- **Phishing-specific triage**
    - **PhishTool** — consolidates email metadata, attachments, URLs, and integrates VirusTotal; extracts hashes, strings, SMTP hops, reply/return-path etc.

## Typical analysis workflow (practical steps)

1. **Capture raw email** (header + body) and **save attachments** without opening them.
2. **Extract and record header artifacts** (sender, IPs, hops, reverse DNS, reply-to, timestamps).
3. **Extract URLs** from the HTML/text (do not browse them in your normal browser). Feed those to urlscan.io or a sandbox.
4. **Save attachment(s)** and compute SHA256 (record filename + hash).
5. **Lookup hashes/URLs/domains in VirusTotal / Talos / ipinfo / urlscan**. Record findings and screenshots.
6. If needed, **submit the file/URL to a malware sandbox** for dynamic behaviour and IOCs.
7. **Document**: all artifacts, tool outputs, verdict (malicious/benign/unknown), and suggested remediation (block domain, delete message, isolate host).
8. **Classify** the phishing type (phish, credential harvesting, whaling, business email compromise) and tag as needed.

## Notes on classification and reporting

- Use classification codes (e.g., whaling, credential phishing, malware delivery) to prioritize response — not every case is the same.
- Keep a tamper-proof copy of the raw email and all hashes/screenshots for incident records.

## Quick one-line checklist you can copy/paste

- Raw header, sender email, sender IP(s), PTR, reply-to, recipients, subject, timestamp; all URLs + root domains; attachment names + SHA256; screenshots + tool outputs (urlscan/VirusTotal/ipinfo); final verdict & remediation.
