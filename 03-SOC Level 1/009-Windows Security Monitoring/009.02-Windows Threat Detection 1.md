# Windows Threat Detection 1

## High-level concept

- **Initial Access** = the attacker’s “front door” into an environment. Two broad families:
    - **Exposed services** (internet-facing services: RDP, SSH, HTTP, SMTP, MS-SQL).
    - **User-driven** (phishing, removable media, social engineering) — exploits human behaviour to bypass perimeter controls.

## Exposed services (what to worry about)

- Commonly exposed services: RDP, VNC, SSH, web apps, mail servers.
- Risks: automated scanning, credential brute force, exploitation of unpatched apps.
- MITRE examples: **T1133** (External Remote Services), **T1190** (Exploit Public-Facing Application).
- Outcome: fast compromise (RDP called out as very common vector for ransomware).

## Initial Access via RDP — key points for SOC

- **Risk:** Millions of RDP hosts exposed; weak/default passwords get compromised quickly.
- **Detection priorities / artifacts:**
    - Windows Security logs:
        - **4625** = failed logon attempts → hunt for bursts from same source or username (brute force).
        - **4624** = successful logon → confirm account used for initial access.
        - **Logon types**: **3** (network), **10** (remote/interactive/RDP) — filter these.
        - **Source IP** field → external IPs performing attempts.
        - Capture and track **Logon ID** from successful 4624 (use to correlate subsequent activity).
    - Use Sysmon to link actions: find processes/events with the same **Logon ID** to see what the threat actor executed after RDP access.
- **Hunting checklist:** spike of 4625s → find corresponding 4624s → extract Logon ID → search Sysmon for that Logon ID.

## Initial Access via Phishing — key points

- **State:** Phishing remains highly effective; attackers increasingly use conversational LLMs to craft lures.
- **Two common payload types:**
    1. **Binary attachments** (.exe, .scr, .com, .cpl, etc.) — often disguised with double extensions or icons; Windows hides extensions by default which aids deception.
    2. **LNK shortcuts** that execute commands (PowerShell, VBScript, etc.) — look like benign shortcuts but contain malicious command in the Target field.
- **Why LNKs are dangerous:** They often leave the execution parent as `explorer.exe`, obscuring the user-action origin; however, you can detect them by finding prior file creation events (LNK appearing in Downloads).
- **Sysmon-based detection (example sequence):**
    1. Sysmon **Event ID 1**: browser launched (`msedge.exe` / `chrome.exe`) with ParentImage `Explorer.exe`.
    2. Sysmon **Event ID 11**: file downloaded to `C:\Users\<user>\Downloads\invoice.zip`.
    3. Event ID 11: archive extraction (`7zG.exe` / Explorer) producing `invoice.pdf.exe` or `VisitOurWebsite!.lnk`.
    4. Event ID 1: user double-clicks the payload (`invoice.pdf.exe` or LNK→PowerShell) — ParentImage `Explorer.exe`.
- **Hunting tips:** search for unusual downloads, double-extension filenames, LNK files in Downloads, Explorer spawning PowerShell/cmd with encoded commands or remote download behavior (WebClient/Invoke-WebRequest/Start-Process).

## Initial Access via USB (removable media)

- **Risk:** Bypasses network controls; worms and USB-delivered malware still used (e.g., Raspberry Robin-style activity).
- **Common patterns on USB-delivered attacks:**
    - Hidden legitimate files + malicious `RECOVERY.lnk`.
    - Executables masquerading as folders (`Photos.exe` with folder icon).
    - Double-extension files (`photo.jpg.exe`).
- **Detection signals:**
    - Process Image paths referencing external drive letters (e.g., `E:\malware.exe`) in process creation events.
    - File creation events on mount (new LNK/exe appearing on removable volumes).
    - Same Sysmon sequence as phishing once user executes the file (Explorer → executable).
- **Hunting tips:** monitor for processes launched from non-standard volumes (drive letters other than `C:\`), new autorun-like artifacts (even though modern Windows blocks autorun, many attackers rely on user action).

## Practical SOC playbook (quick checklist)

- Monitor Security logs for:
    - **Burst of 4625** from single IP or many usernames → possible brute force.
    - **4624** with logon type **10** → confirm RDP interactive access.
    - Capture **Logon ID** and correlate with Sysmon events.
- Deploy/inspect Sysmon:
    - Event ID **1** (process create) and **11** (file creation) as primary signals for download/unpack/execute chains.
    - Alert on `explorer.exe` spawning PowerShell/encoded command lines or downloading executables.
    - Alert on processes launched from removable-drive paths (E:, F:, etc.).
- Hunt for artifacts of deception:
    - LNK files in user Download or Desktop folders.
    - Files with hidden or double extensions; unusual icons for executables.
    - Browser -> archive -> unpack -> execution chains.
- Remediation steps on detection:
    - Isolate the host, collect logs (Security + Sysmon), extract Logon ID and timeline, block source IPs and accounts, reset credentials, and scan for persistence.

## Notable real-world context (brief)

- Ransomware groups commonly use exposed RDP and phishing; threat reports show multiple groups (e.g., Medusa, Akira) using these vectors across campaigns.
- Phishing volumes and sophistication are increasing — defenders must monitor both network-facing exposure and user-driven vectors.
