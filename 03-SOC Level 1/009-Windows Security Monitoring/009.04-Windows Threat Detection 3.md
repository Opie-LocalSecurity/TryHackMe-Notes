# Windows Threat Detection 3

## Big picture

- **C2 purpose:** provide a persistent, remote channel allowing attackers to run commands, transfer files, and maintain access after initial compromise.
- **When C2 isn‚Äôt needed:** interactive access methods (e.g., RDP) allow direct command execution but are short-lived; attackers usually establish C2 anyway to survive session termination.

## C2 delivery patterns

- **Immediate connecter:** payload (e.g., phishing attachment) itself opens the C2 channel on execution (simple, common).
- **Dropper + loader:** initial payload downloads and launches a stealthier C2 binary (keeps persistence if initial file is deleted).
- **Examples cited:** Cobalt Strike and real-world ransomware/APT campaigns (loader ‚Üí persistent C2).

## Why persistence matters

- Maintains long-term access for botnets, espionage, lateral movement, and ransomware preparation.
- Attackers may add victims into botnets, spy (months-long presence), or use one host to penetrate a whole AD environment.

## Common Windows persistence techniques (summary + detection signals)

### Backdoored user accounts

- **Actions:** create new local users or reset existing passwords; add them to privileged groups (Administrators, Remote Desktop Users).
- **Commands/tools:** `net user`, `New-LocalUser`, `net localgroup` / `Add-LocalGroupMember`.
- **Key Event IDs to monitor:**
    - **4720** ‚Äî user account created
    - **4732** ‚Äî user added to a privileged group
    - **4724** ‚Äî password reset
- **Detection approach:** investigate *who* performed the action, source IP/time of creator‚Äôs login, other suspicious events in that session.

### Windows Services (system-wide, starts at boot)

- **Action/command example:** `sc create "BadService" binpath= "C:\malware.exe" start= auto`
- **Detection signals:**
    - Sysmon event: **1** ‚Äî execution of `sc.exe` (or parent process that created it)
    - Security event: **4697** ‚Äî service installation
    - System event: **7045** ‚Äî service created (alternative)
    - Process tree: suspicious child process of `services.exe`
- **Why used:** runs with SYSTEM privileges and persists across reboots.

### Scheduled Tasks (flexible, powerful)

- **Action/command example:** `schtasks /create /tn "BadTask" /tr "C:\malware.exe" /sc onstart /ru System`
- **Detection signals:**
    - Sysmon **1** ‚Äî execution of `schtasks.exe`
    - Security **4698** ‚Äî scheduled task creation
    - Process parent: `svchost.exe [...] -s Schedule`
- **Why used:** easy to configure, widely abused by many APTs.

### Run Keys & Startup folder (per-user persistence)

- **Startup folder path:** `%AppData%\Microsoft\Windows\Start Menu\Programs\Startup\` (per-user) or `C:\ProgramData\...\StartUp` (all users)
- **Run keys:** `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` and `HKLM\...\Run`
- **Examples:**
    - Copy to Startup folder
    - `reg add "HKCU\...\Run" /v BadKey /t REG_SZ /d "C:\malware.exe"`
- **Detection signals:**
    - Sysmon **11** ‚Äî file create in Startup folder
    - Sysmon **13** ‚Äî registry value creation/modification (Run keys)
    - Note: processes launched via Startup often have `explorer.exe` parent ‚Üí noisy false positives

## Practical SOC L1/L2 checklist (prioritized)

1. **Monitor the three high-value event groups:**
    - Account activity: **4720, 4724, 4732**
    - Service/task creation: **4697, 4698, 7045**
    - Sysmon process/registry/file events: **1, 11, 13**
2. **Alert triage logic:**
    - Correlate *who* created the object with source IP and recent logons.
    - If admin-privileged process (sc/schtasks) spawned from non-admin user or unusual parent ‚Üí escalate.
    - File creation in Startup folder + unexpected explorer.exe child ‚Üí investigate user activity and file hash.
3. **Hunt for lateral stages:**
    - New privileged accounts used for RDP logins.
    - Task/service binaries stored in non-standard locations (e.g., `C:\Temp`, `%APPDATA%`).
4. **Containment tips:**
    - Disable suspicious accounts, but first capture session/log context.
    - Remove malicious services/tasks and quarantine binaries after confirming.
    - Rotate credentials and examine AD for further persistence.

## Real-world notes & outcomes

- Attackers often chain methods: initial access ‚Üí dropper ‚Üí C2 binary ‚Üí create persistent service/task or backdoor users ‚Üí lateral movement ‚Üí data exfiltration/ransomware.
- Ransomware cases typically show multi-week campaigns where persistence enables full network compromise (AD-targeting leads to broad impact).

## Quick reference (one-liners)

- **Watch:** 4720, 4724, 4732, 4697, 4698, 7045, Sysmon 1/11/13.
- **Look for:** `sc.exe`, `schtasks.exe`, new Run-key registry writes, files in Startup folders, unusual privileged user creation or password resets.
- **Tools/UI:** `services.msc`, `taskschd.msc`, `lusrmgr.msc`, `regedit` for investigative follow-up.
