# Data Exfiltration Detection

## What it is & why it’s done

- **Definition:** Unauthorized transfer of data from an org to attacker-controlled infrastructure (via insiders or compromised systems).
- **Motives:**
    - Financial fraud/monetization
    - Espionage of IP/trade secrets
    - Ransomware & extortion (steal-then-encrypt)
    - Disruption/sabotage of operations & reputation
    - Recon for follow-on attacks

## Real-world actors & typical exfil paths

- **APT29 (Cozy Bear):** HTTPS over legit domains to blend in.
- **FIN7:** Embed stolen data in **HTTP POST** to C2.
- **Lunar Spider (Zloader):** Long-lived intrusions, **encrypted C2** and staged transfers.
- **DarkSide:** **Dual extortion**—exfil + encrypt.
- **APT10 (Cloud Hopper):** **Cloud-to-cloud** (MSP/tenant APIs).

## Common exfil phases

1. **Discovery/Collection:** Find sensitive files/DBs.
2. **Staging/Preparation:** Aggregate → compress/encrypt/encode (zip/7z/rar, base64, stego).
3. **Transport:** Network (HTTP(S), DNS, FTP, ICMP), removable media, cloud storage, covert channels.
4. **C2 Coordination:** Orchestrate and confirm receipt.

## Cross-domain detection mindset

- **Correlate, don’t single-alert.** Tie together: source user/host, destination, data volume, process/command line, and corroborating logs.
- **Primary telemetry:** Proxy/Web GW, Firewall/NetFlow, DNS, EDR/Sysmon, Cloud audit logs, Mail/Collab audits.

## Technique → Indicators → Where to look

- **Network-based:** HTTP(S) uploads (S3/Azure/webmail), FTP/SFTP/SCP, DNS tunneling, ICMP/covert TCP/UDP
    - *Signals:* Large POSTs, big egress to one IP/ASN, spikes in outbound flows, long/odd DNS names, TXT volume
    - *Sources:* Proxy, NGFW/NetFlow, DNS logs
- **Host-based:** PowerShell `Invoke-WebRequest`, `curl/wget`, `rclone`, `awscli`, archive creation, USB usage
    - *Signals:* Suspicious process/create/network events, object access logs
    - *Sources:* Sysmon/EDR (EventID 1/3/11), Windows 4663/4656, Linux auditd/history
- **Cloud exfil:** S3 `PutObject`/multipart, Azure Blob/GCS uploads, M365 Drive/SharePoint external shares
    - *Signals:* Unusual service accounts/IPs, off-hours spikes, sharing to externals
    - *Sources:* CloudTrail / Azure Activity / GCP Audit / Storage access logs
- **Covert & encoding:** DNS/base32/64, stego, many small/chunked requests (low-and-slow)
    - *Signals:* High-entropy strings, regular intervals, many small POSTs
- **Insider & collab tools:** Slack/Teams/Dropbox/Drive/Box
    - *Signals:* Mass downloads + external shares
    - *Sources:* App audit logs, mail logs

## DNS Tunneling — Detection Playbook

**Why DNS:** Ubiquitous, usually allowed; payload can hide in subdomains/TXT.

**Wireshark workflow (pcap: `dns_exfil.pcap`):**

- Show DNS: `dns`
- Queries only (no responses): `dns.flags.response == 0`
- Long queries: `dns && frame.len > 70`
- Pivot to suspicious domain: `dns && dns.qry.name contains <REDACTED>`

**Indicators:**

- Very **long FQDNs** (>60–100 chars), **high-entropy** subdomains (base32/64, digits, `=`), many **TXT**/NULL, frequent **NXDOMAIN**, **regular intervals**, many queries to **one domain**.

**Splunk workflow:**

- Scope logs: `index=data_exfil sourcetype=DNS_logs`
- Top talkers: `index="data_exfil" sourcetype="DNS_logs" | stats count by src_ip`
- Noisy query strings: `index="data_exfil" sourcetype="dns_logs" | stats count by query | sort -count`
- Long queries: `index="data_exfil" sourcetype="DNS_logs" | where len(query) > 30`

**Outcome:** Identify compromised hosts sending chunked data to a single external domain.

## FTP Exfiltration — Detection Playbook

**Why FTP:** Simple, legacy, sometimes exposed/misconfigured; can use non-standard ports.

**Wireshark (pcap: `ftp-lab.pcap`):**

- Isolate control/data: `ftp || ftp-data`
- Credentials in cleartext: `ftp.request.command == "USER" || ftp.request.command == "PASS"`
- Upload ops: `ftp contains "STOR"` → **Follow TCP Stream** for file names/paths
- Hunt sensitive types: `ftp contains "csv"`
- Large control frames (proxy for big transfers): `ftp && frame.len > 90` → inspect streams

**Indicators:**

- Guest/weak creds, repeated **STOR**, large PASV data channels to **unusual IPs**, off-hours bursts, suspicious filenames (e.g., `.csv`, `.pdf`).

**Outcome:** Confirm sensitive document uploads to external IPs.

## HTTP Exfiltration — Detection Playbook

**How it’s abused:** Big **POST** bodies, **GET** with encoded params, **custom headers**, **chunked/multipart**, **HTTPS** to mask payloads, uploads to cloud/CDN or reputable domains.

**Splunk (HTTP logs):**

- Scope: `index="data_exfil" sourcetype="http_logs"`
- Focus POSTs: `index="data_exfil" sourcetype="http_logs" method=POST`
- Size & domain profiling: `index="data_exfil" sourcetype="http_logs" method=POST | stats count avg(bytes_sent) max(bytes_sent) min(bytes_sent) by domain | sort - count`
- Pull large uploads: `index="data_exfil" sourcetype="http_logs" method=POST bytes_sent > 600 | table _time src_ip uri domain dst_ip bytes_sent | sort - bytes_sent`

**Wireshark (pcap: `http_lab.pcap`):**

- All HTTP: `http`
- POSTs: `http.request.method == "POST"`
- Size thresholds: `http.request.method == "POST" and frame.len > 500` → then `> 750`
- Inspect content: **Follow HTTP/TCP stream** (look for documents, archives, base64).

**Indicators:**

- Rare domains + **large POSTs**, beaconing followed by upload, bursty multipart/chunked series, odd headers (`X-*` carrying base64), uploads to storage endpoints.

**Outcome:** Single large POST aligns with logs → confirms secret doc exfil to external IP.

## ICMP Exfiltration — Detection Playbook

**How it’s abused:** Encode file chunks in **ICMP Echo** payloads (type 8/0), possibly fragmented; often base64/hex and periodic.

**Wireshark (pcap: `icmp_lab.pcap`):**

- All ICMP: `icmp`
- Echo Requests: `icmp.type == 8`
- Large payloads: `icmp.type == 8 and frame.len > 100`

**Indicators:**

- Persistent ICMP to one external IP, payload sizes **well above ping norms** (~74 bytes total typical), high-entropy data, even spacing, unusual types/codes.

**Outcome:** Flag oversized, periodic ICMP as likely tunneling.

## Fast Triage Checklist (SOC L1)

1. **Scope the host/user:** Who, where, and when (off-hours?).
2. **Quantify egress:** Total bytes, top destinations, single IP/ASN concentration.
3. **Identify channel:** DNS/HTTP/FTP/ICMP/Cloud; confirm with **process lineage** (EDR).
4. **Artifact check:** Archives created? `rclone/awscli/curl` executions? USB events?
5. **Cloud & collab:** External shares, abnormal service accounts, access from new IPs.
6. **Corroborate across logs:** Proxy/Firewall/NetFlow + DNS + Sysmon/EDR + Cloud audit.
7. **Preserve evidence:** PCAPs, logs, process trees, hashes; block destinations; rotate creds.

## Key Takeaways

- Exfil is **multi-stage** and often hides in **legitimate protocols/services**.
- **Correlation beats single alerts**—link size, frequency, destinations, and host activity.
- Have **ready-made filters/queries** for DNS, FTP, HTTP, and ICMP to accelerate triage.
- Focus on **outliers**: long/high-entropy strings, unusual destinations, large/periodic egress, suspicious tools/commands, and cloud sharing anomalies.
