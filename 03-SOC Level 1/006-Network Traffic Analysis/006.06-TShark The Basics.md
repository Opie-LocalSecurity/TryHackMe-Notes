# TShark: The Basics

TShark is a powerful, open-source command-line network analyzer developed by the Wireshark team. It bridges the gap between the detailed analysis of Wireshark and the lightweight, scriptable nature of `tcpdump`, making it ideal for automation and deep packet inspection.

## 1. Essential Supplemental Tools

Because TShark is CLI-based, its output is frequently piped into standard Linux utilities to refine data:

- **`capinfos`**: Provides a high-level summary of a capture file (duration, packet count, hashes).
- **`grep` / `cut` / `awk`**: Used for pattern searching and isolating specific data fields.
- **`uniq` / `nl`**: Useful for filtering duplicates and counting the number of filtered results.

## 2. Core Parameters & Operations

| **Parameter** | **Function** | **Context** |
| --- | --- | --- |
| `-D` | **List Interfaces** | Shows available sniffing points (e.g., eth0, lo). |
| `-i [index]` | **Select Interface** | Chooses a specific interface for live capture. |
| `-r [file]` | **Read File** | Processes an existing `.pcap` or `.pcapng` file. |
| `-w [file]` | **Write File** | Saves sniffed or filtered traffic to a file. |
| `-c [count]` | **Packet Limit** | Stops operation after a specific number of packets. |
| `-V` | **Verbosity** | Provides full packet details (mimics Wireshark's Detail Pane). |
| `-x` | **Hex/ASCII Dump** | Shows raw packet bytes. |

## 3. Capture Conditions (Autostop & Ring Buffers)

TShark can manage live captures based on specific triggers to prevent storage exhaustion or to segment data.

- **Autostop (`a`)**: Stops the capture after a condition is met (e.g., `a duration:10` for 10 seconds or `a filesize:1000` for 1MB).
- **Ring Buffer (`b`)**: Creates an infinite loop of files. Once the file limit is reached, it overwrites the oldest file (e.g., `b files:5` keeps only the last 5 files).

## 4. Filtering Logic: Capture vs. Display

TShark utilizes two distinct filtering layers. It is critical to use single quotes (`'filter'`) to prevent shell expansion issues.

### **Capture Filters (`f`)**

- **Purpose**: Applied *during* live capture to limit what is saved to disk.
- **Syntax**: Berkeley Packet Filter (BPF) syntax.
- **Examples**:
    - `tshark -f "host 10.10.10.10"`
    - `tshark -f "port 80"`
    - `tshark -f "icmp"`

### **Display Filters (`Y`)**

- **Purpose**: Applied *after* or *during* reading to change what is visible in the output without deleting the underlying data.
- **Syntax**: Standard Wireshark display filter syntax.
- **Examples**:
    - `tshark -r file.pcap -Y 'ip.addr == 192.168.1.1'`
    - `tshark -r file.pcap -Y 'http.response.code == 200'`
    - `tshark -r file.pcap -Y 'dns.qry.type == 1'` (A records)

## 5. Practical Methodology

1. **Survey**: Use `capinfos` to understand the scope of a capture file.
2. **Isolate**: Use `r` with `Y` to find specific traffic.
3. **Count**: Pipe to `nl` (e.g., `tshark -r demo.pcap -Y 'http' | nl`) to see the true count of filtered packets rather than the original frame numbers.
4. **Export**: Use `w` to save a subset of suspicious packets for further investigation.
