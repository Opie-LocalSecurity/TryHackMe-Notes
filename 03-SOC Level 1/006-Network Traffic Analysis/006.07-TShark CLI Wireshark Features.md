# TShark: CLI Wireshark Features

## 1. Core Statistics and Visualization

TShark uses the `-z` parameter to generate statistics. Use `-q` to suppress individual packet details and only display the summary.

- **Protocol Hierarchy:** Provides a tree view of protocol distribution.
    - `tshark -r file.pcap -z io,phs -q`
- **Packet Lengths:** Visualizes distribution of packet sizes to spot anomalies (e.g., massive data exfiltration or tiny keep-alive heartbeats).
    - `tshark -r file.pcap -z plen,tree -q`
- **Endpoints & Conversations:** Identifies unique hosts and the traffic flow between them.
    - **Endpoints:** `z endpoints,<type> -q` (Types: `eth`, `ip`, `ipv6`, `tcp`, `udp`)
    - **Conversations:** `z conv,<type> -q`
- **Expert Info:** Displays automated Wireshark insights (e.g., TCP retransmissions, SYN requests).
    - `tshark -r file.pcap -z expert -q`

## 2. Protocol-Specific Statistics

Used to drill down into specific network layers or services:

| **Protocol** | **Command Parameter** | **Purpose** |
| --- | --- | --- |
| **IPv4/IPv6** | `-z ptype,tree` | General IP protocol distribution. |
| **Hosts** | `-z ip_hosts,tree` | List of all unique IP addresses. |
| **Destinations** | `-z dests,tree` | Outgoing traffic, including ports/services. |
| **DNS** | `-z dns,tree` | Summary of RCODEs and query types. |
| **HTTP** | `-z http,tree` | Status codes (200 OK, 404, etc.) and load. |
| **HTTP Req** | `-z http_req,tree` | Breakdown of specific HTTP requests. |

## 3. Data Extraction and Object Recovery

TShark can reconstruct streams and extract files directly from the CLI.

### Follow Stream

Reassembles the payload of a specific conversation (Index starts at 0).

- **TCP:** `z follow,tcp,ascii,0 -q`
- **HTTP:** `z follow,http,ascii,1 -q`

### Export Objects

Extracts files sent over unencrypted protocols (`HTTP`, `SMB`, `TFTP`, `DICOM`, `IMF`).

- **Usage:** `-export-objects <protocol>,<destination_folder> -q`

### Credentials

Automatically detects cleartext credentials in protocols like `FTP`, `HTTP`, and `POP`.

- **Usage:** `z credentials -q`

## 4. Advanced Filtering and Field Extraction

### Comparison Operators

- **Contains:** Case-sensitive search for a specific value within a field (e.g., `http.server contains "Apache"`).
- **Matches:** Case-insensitive search using **Regular Expressions** (e.g., `http.request.method matches "(GET|POST)"`).

### Custom Field Extraction

Use the `-T fields` flag to output specific data points instead of full packet summaries.

- **`e <field>`**: Specific field to extract (e.g., `ip.src`, `dns.qry.name`).
- **`E header=y`**: Includes headers in the output for readability.

## 5. Security Use Cases & Post-Processing

For high-volume analysis, combine TShark with Linux utilities (`awk`, `sort`, `uniq`) to identify patterns.

- **Extracting Hostnames (DHCP):**
    
    `tshark -r file.pcap -T fields -e dhcp.option.hostname | awk NF | sort | uniq -c | sort -r`
    
- **Extracting DNS Queries:**
    
    `tshark -r file.pcap -T fields -e dns.qry.name | awk NF | sort | uniq -c | sort -r`
    
- **Extracting User Agents (HTTP):**
    
    Useful for spotting automated scanners like `sqlmap`, `Wfuzz`, or `Nmap`.
    
    `tshark -r file.pcap -T fields -e http.user_agent | awk NF | sort | uniq -c | sort -r`
