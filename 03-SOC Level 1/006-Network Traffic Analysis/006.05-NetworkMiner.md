# NetworkMiner

## **Purpose / Scope**

- NetworkMiner = open-source Network Forensic Analysis Tool (NFAT). Primary use: passive network forensics (Windows first; runs on Linux/macOS/FreeBSD).
- Two main workflows: (1) live passive sniffing (Windows-only, limited), (2) offline PCAP parsing & extraction (primary value).

## Key Capabilities (what it gives you quickly)

- Host/context extraction: IP, MAC, hostnames, OS fingerprinting (uses Satori + p0f), open ports, session counts.
- File extraction: images, HTML, email attachments, reconstructed file paths.
- Credential & parameter extraction: HTTP/FTP/IMAP/SMTP cookies/requests, Kerberos/NTLM hashes, RDP cookies, SQL credentials.
- Keyword/cleartext extraction: searchable cleartext strings and configurable keywords.
- Session/DNS analysis: lists sessions and DNS queries with timestamps, TTLs, answers.
- Anomaly detection: limited, includes a few exploit/spoofing detections (e.g., EternalBlue heuristics).
- Case management: load multiple PCAPs, view metadata, export/open extracted artifacts.

## Operating Modes & Best Practice

- **Sniffer Mode**: Available but not robust — Windows-only and not recommended as the primary sniffer.
- **PCAP Parsing Mode**: Primary recommended mode — fast “low-hanging fruit” overview before deeper analysis with other tools (e.g., Wireshark).

Best practice: record traffic offline → quick overview & extraction with NetworkMiner → deep packet/protocol/payload analysis in Wireshark/tcpdump as needed.

## UI / Major Views (what to expect)

- **Landing / File menu**: load PCAPs, drag & drop, PCAP-over-IP supported.
- **Case panel**: manage loaded PCAPs, view metadata, reload/remove.
- **Hosts**: sortable host list with OS, MAC vendor, ports, packets, sessions.
- **Sessions**: client/server, ports, protocol, frame number, start time; supports four search modes (ExactPhrase, AllWords, AnyWord, RegEx).
- **DNS**: detailed DNS query/answer listings (Alexa Top1M lookup in Pro).
- **Credentials**: extracted hashes/creds, easy copy/export; integrates with cracking tools externally.
- **Files / Images / Messages / Parameters / Keywords**: extracted artifacts and metadata; right-click actions, built-in viewers.
- **Anomalies**: lightweight detections for known exploit patterns/spoofs.

## Strengths (Pros)

- Rapid host/context discovery and OS fingerprinting.
- Easy and effective file extraction from PCAPs.
- Automated credential and parameter extraction (saves manual searching).
- Quick keyword discovery and overall “map” of network activity — excellent for initial triage.

## Limitations (Cons)

- Not reliable as a primary live sniffer.
- Poor scalability for very large PCAPs (not optimal for heavy-duty packet-level triage).
- Limited filtering, packet decoding, payload and deep protocol analysis.
- Not designed for manual in-depth packet-by-packet investigation — use Wireshark for that.

## Comparison: NetworkMiner vs Wireshark (high-level)

- NetworkMiner = quick overview, data extraction, host categorization, OS fingerprinting.
- Wireshark = in-depth packet decoding, protocol analysis, advanced filtering, payload inspection, statistics.
- Recommended workflow: NetworkMiner for quick extraction/triage → Wireshark for deep analysis.

## Version Differences / Notable Feature Changes

- **v2+**: improved MAC-address correlation, expanded parameter extraction, better host correlation.
- **≤ v1.6**: more granular frame-level and cleartext views (cleartext tab present in v1.6, removed later), more detailed per-frame controls.
- Result: later versions add broader parameter/host features; earlier versions expose different low-level frame/cleartext views—both have use cases depending on investigative needs.

## Practical Outcomes / How investigators use it

- Quickly identify suspicious hosts, extract files and credentials, find indicators (malicious domains, file artifacts), and collect evidence to prioritize deeper analysis.
- Good for incident responders and law enforcement as initial triage and artifact collection tool.

## Quick Reference / Checklist for Triage

- Load PCAP → check **Hosts** for unknown OS/MACs and open ports.
- Review **Sessions** and **DNS** for suspicious endpoints or domain lookups.
- Inspect **Credentials** and **Parameters** for leaked credentials or plaintext secrets.
- Extract files/images from **Files/Images** and hash / submit as needed (Pro adds hash/OSINT lookup).
- Use **Keywords** to search for IOCs, then export interesting frames to Wireshark for deep dive.
