# Wireshark: The Basics

## What it’s for

- **Network troubleshooting:** find failure points, congestion.
- **Security triage:** spot rogue hosts, odd ports, suspicious flows.
- **Protocol learning:** inspect headers, codes, payloads.
- **Not an IDS:** passive packet reader; value depends on analyst skill.

## GUI essentials (start screen)

- **Toolbar:** capture, filter, sort, summarize, export, merge.
- **Display Filter Bar:** main query box for display filters.
- **Recent Files:** quick reopen.
- **Capture Filters & Interfaces:** pick NICs (e.g., `lo`, `eth0`, `ens33`) and pre-capture BPF filters.
- **Status Bar:** profile + counts (total vs displayed).

## Loading & viewing PCAPs

- Open via **File / drag-drop / double-click**.
- Three panes:
    - **Packet List:** per-packet summary (src/dst, proto, info).
    - **Packet Details:** decoded protocol tree.
    - **Packet Bytes:** hex + ASCII; highlights follow selection in Details.

## Packet colouring

- Speeds anomaly/protocol spotting.
- **Permanent rules:** **View → Coloring Rules** (saved in profile).
- **Temporary rules:** **View → Conversation Filter** or right-click.
- Toggle with **View → Colorize Packet List**.

## Capturing

- **Blue shark:** start capture.
- **Red square:** stop.
- **Green arrow:** restart.
- Status bar shows interface + packet count.

## File ops

- **Merge pcaps:** **File → Merge** (then **File → Save As** to keep the merged result).
- **Capture file properties:** **Statistics → Capture File Properties** (hash, time, interface, comments, stats).

## Dissection (OSI-aligned drill-down)

Typical layers shown for a packet:

1. **Frame (L1):** capture metadata (time, length).
2. **Ethernet/MAC (L2):** src/dst MAC.
3. **IP (L3):** src/dst IPv4/IPv6, TTL, flags.
4. **Transport (L4):** TCP/UDP ports, flags, seq/ack.
    - **Protocol Errors/Reassembly:** TCP segment reassembly notes.
5. **Application (L7):** protocol specifics (HTTP/FTP/SMB…).
6. **Application Data:** reconstructed payload (when available).

## Navigating packets

- **Packet numbers:** unique IDs in the file to anchor analysis.
- **Go to:** **Go → Go to Packet** (by number; next/previous in conversation).
- **Find:** **Edit → Find Packet**
    - Input types: **Display filter**, **Hex**, **String**, **Regex** (case toggle).
    - Scope: **List / Details / Bytes** (choose where the data actually appears).
- **Mark/Unmark:** right-click or **Edit**; shows as black rows (resets when file closes).
- **Packet comments:** persistent annotations stored in the pcap.
- **Export packets:** **File → Export Specified Packets** (share only the scoped subset).
- **Export Objects:** pull transferred files for **DICOM/HTTP/IMF/SMB/TFTP**.
- **Time display:** **View → Time Display Format** (use UTC or preferred format).
- **Expert Info:** **Analyze → Expert Information**
    - Severities: **Chat (blue)**, **Note (cyan)**, **Warn (yellow)**, **Error (red)**
    - Common groups: **Checksum**, **Comment**, **Deprecated**, **Malformed**.

## Filtering (core workflow)

- **Capture filters** (pre-capture, BPF syntax) vs **Display filters** (post-capture, Wireshark syntax).
- Golden rule: **“If you can click it, you can filter/copy it.”**

### Display-filter actions

- **Apply as Filter:** right-click or **Analyze → Apply as Filter** (immediate include/exclude).
- **Conversation Filter:** show only packets of the selected flow (IP/port 5-tuple).
- **Colorize Conversation:** highlight related packets without hiding others (**View → Colorize Conversation**; reset via same menu).
- **Prepare as Filter:** stage query in the bar (chain with **…and/or…**) before applying.
- **Apply as Column:** add any field as a column (great for scanning values across packets).
- **Follow Stream:** **Analyze → Follow {TCP|UDP|HTTP} Stream**
    - Reassembles bidirectional payload; client=red, server=blue.
    - Auto-applies a conversation filter; clear via the **X** in the filter bar.

## Quick recall tips

- **Counts matter:** status bar shows **Displayed / Total**—watch it when filtering.
- **Profiles:** save colouring rules, columns, and prefs to keep a consistent workspace.
- **Workflow pattern:** *Locate (Find/Expert) → Scope (Apply/Conversation) → Inspect (Details/Bytes/Follow) → Extract (Export packets/objects) → Note (Comments/Marks).*
